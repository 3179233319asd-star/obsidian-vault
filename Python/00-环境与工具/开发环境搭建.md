# 开发环境搭建

## Python 安装

官网 [python.org](https://www.python.org/downloads/) 下载，**务必勾选 "Add Python to PATH"**。

```bash
python --version   # 验证安装
pip --version
```

> Mac 用户注意：系统自带的 `python` 可能是 Python 2，请统一用 `python3`。

---

## pip 包管理器

```bash
pip install requests              # 安装
pip install requests==2.31.0      # 指定版本
pip install --upgrade requests    # 升级
pip uninstall requests            # 卸载
pip list                          # 已安装列表
pip show requests                 # 查看某个包的详情
pip freeze > requirements.txt     # 导出依赖
pip install -r requirements.txt   # 批量安装
```

### 版本约束符号

| 符号 | 含义 | 示例 |
|------|------|------|
| `==` | 精确版本 | `requests==2.31.0` |
| `>=` | 最低版本 | `requests>=2.28` |
| `~=` | 兼容版本 | `requests~=2.31.0` 等价于 `>=2.31.0, <2.32.0` |

### 配置清华镜像（国内提速）

```bash
# 临时使用
pip install requests -i https://pypi.tuna.tsinghua.edu.cn/simple

# 永久配置
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
```

常用国内镜像源：

| 镜像源 | 地址 |
|--------|------|
| 清华大学 | `https://pypi.tuna.tsinghua.edu.cn/simple` |
| 阿里云 | `https://mirrors.aliyun.com/pypi/simple/` |
| 中科大 | `https://pypi.mirrors.ustc.edu.cn/simple/` |

## 为什么需要环境管理？

不同项目可能依赖同一个库的**不同版本**：

- 项目 A 需要 `requests==2.28.0`，Python 3.10
- 项目 B 需要 `requests==2.31.0`，Python 3.12

如果在全局环境安装，版本会互相冲突。**环境管理工具为每个项目创建独立的 Python 运行环境，互不干扰。**

```
环境管理工具
├── 项目 A 的虚拟环境（Python 3.10, requests 2.28.0, flask 2.3.0）
├── 项目 B 的虚拟环境（Python 3.12, requests 2.31.0, django 4.2.0）
└── 项目 C 的虚拟环境（Python 3.11, numpy 1.26.0）
```

两种主流工具：**Conda**（数据科学标配）和 **uv**（新一代，强烈推荐）。

## Conda 环境管理

数据科学/AI 领域标配。能管理 Python 版本 + 非 Python 系统库（CUDA 等）。

### 安装

推荐安装 **Miniconda**（~80 MB），而非 Anaconda（~3 GB，预装太多用不上的包）。

- 官方：[docs.conda.io](https://docs.conda.io/en/latest/miniconda.html)
- 清华镜像：[mirrors.tuna.tsinghua.edu.cn/anaconda/miniconda](https://mirrors.tuna.tsinghua.edu.cn/anaconda/miniconda/)

Windows 安装后需要在 PowerShell 中初始化：

```powershell
conda init powershell
# 重启 PowerShell 后生效
```

### 配置清华镜像

```bash
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/main
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud/conda-forge
conda config --set show_channel_urls yes
```

### 环境管理

```bash
# 创建环境（指定 Python 版本）
conda create -n myproject python=3.12

# 创建环境并同时安装包
conda create -n data_env python=3.11 numpy pandas jupyter

# 激活 / 退出
conda activate myproject   # 提示符变为 (myproject)
conda deactivate           # 退出当前环境

# 查看所有环境
conda env list

# 克隆环境
conda create -n myproject_copy --clone myproject

# 删除环境
conda env remove -n myproject
```

### 包管理

```bash
conda install numpy pandas          # 安装
conda install numpy=1.26.0          # 指定版本
conda update numpy                  # 更新
conda remove numpy                  # 卸载
conda list                          # 查看已安装
conda search numpy                  # 搜索可用版本
```

> 混用建议：**先用 conda 装能装的，最后再用 pip 补充**，避免依赖冲突。

### 配置文件管理

```bash
# 导出环境（跨平台，推荐）
conda env export --from-history > environment.yml

# 从配置文件创建环境
conda env create -f environment.yml
```

`environment.yml` 示例：

```yaml
name: data_env
channels:
  - conda-forge
  - defaults
dependencies:
  - python=3.11
  - numpy=1.26.0
  - pandas=2.1.0
  - pip:
    - some-pip-package==1.0.0
```

### Conda 适用场景

| 场景 | 推荐度 | 说明 |
|------|--------|------|
| 数据科学/机器学习 | ⭐⭐⭐⭐⭐ | 原生支持 NumPy、PyTorch、CUDA |
| 需要不同 Python 版本 | ⭐⭐⭐⭐ | 直接管理 Python 版本 |
| 依赖 C/CUDA 系统库 | ⭐⭐⭐⭐⭐ | pip 搞不定的系统级依赖 |
| Web 开发 | ⭐⭐ | 杀鸡用牛刀，uv 更合适 |

## uv —— 新一代工具链（强烈推荐）

Rust 编写，比 pip 快 10-100 倍。**一个工具 = pip + venv + pyenv + poetry**。

### 安装

```bash
# Windows PowerShell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

# Mac
brew install uv

# Linux / Mac
curl -LsSf https://astral.sh/uv/install.sh | sh

# 验证
uv --version
```

### 配置国内镜像

在项目 `pyproject.toml` 中配置（推荐）：

```toml
[[tool.uv.index]]
url = "https://pypi.tuna.tsinghua.edu.cn/simple"
default = true
```

### 管理 Python 版本

uv 可以直接下载和管理 Python，不需要手动安装！

```bash
uv python list                  # 查看可安装版本
uv python install 3.12          # 安装指定版本
uv python pin 3.12              # 固定当前项目 Python 版本（生成 .python-version）
```

### 项目管理（推荐工作流）

```bash
# 1. 创建新项目
uv init my_project
cd my_project

# 项目结构：
# my_project/
# ├── .venv/              # 虚拟环境（自动创建）
# ├── .python-version     # Python 版本锁定
# ├── pyproject.toml      # 项目配置与依赖
# ├── uv.lock             # 精确的依赖锁文件
# ├── README.md
# └── hello.py

# 2. 添加依赖
uv add requests flask
uv add "numpy>=1.26"
uv add --dev pytest ruff mypy     # 开发依赖

# 3. 运行脚本（自动激活虚拟环境！）
uv run python hello.py
uv run pytest

# 4. 同步环境（克隆项目后一条命令搞定）
uv sync

# 5. 移除依赖
uv remove requests
```

### `uv run` 的好处

不需要手动激活虚拟环境，进入项目目录直接用：

```bash
cd project_a
uv run python --version    # Python 3.12.x

cd ../project_b
uv run python --version    # Python 3.10.x
# 自动使用各自项目的虚拟环境！
```

### pip 兼容模式

```bash
uv pip install requests            # 和 pip 一样的语法，快 N 倍
uv pip install -r requirements.txt
uv pip list
uv pip freeze > requirements.txt
```

### 从零到一的完整流程

```bash
# 1. 初始化
uv init my_project && cd my_project

# 2. 加依赖（自动更新 pyproject.toml 和 uv.lock）
uv add requests flask pandas

# 3. 加开发依赖
uv add --dev pytest ruff

# 4. 团队协作——别人克隆后
git clone xxx && cd project
uv sync          # 一条命令装好所有依赖

# 5. 导出传统 requirements.txt（兼容老工具）
uv pip freeze > requirements.txt
```

### uv 快速命令速查

| 命令 | 功能 |
|------|------|
| `uv init` | 初始化项目 |
| `uv add <包名>` | 添加依赖 |
| `uv add --dev <包名>` | 添加开发依赖 |
| `uv remove <包名>` | 移除依赖 |
| `uv sync` | 同步/安装所有依赖 |
| `uv run <命令>` | 在虚拟环境中运行 |
| `uv python install <版本>` | 安装 Python 版本 |
| `uv python pin <版本>` | 固定项目 Python 版本 |
| `uv venv` | 创建虚拟环境 |
| `uv pip install <包名>` | pip 兼容模式 |
| `uv lock --upgrade` | 升级所有依赖 |

## Conda vs uv 对比

| 特性 | Conda | uv |
|------|-------|----|
| 速度 | 慢（秒~分钟） | 极快（毫秒~秒） |
| 开发语言 | Python | Rust |
| 管理 Python 版本 | ✅ | ✅ |
| 管理 C/CUDA 依赖 | ✅ | ❌ |
| 配置文件 | `environment.yml` | `pyproject.toml`（标准） |
| 锁文件 | 第三方 `conda-lock` | 内置 `uv.lock` |
| 磁盘占用 | 较大 | 极小 |
| 适合场景 | 数据科学/AI | Web/通用开发/学习 |

**怎么选？**

- 数据科学/AI，团队用 Conda，需要 CUDA → **Conda**
- Web 开发、通用 Python、追求速度 → **uv**
- 学习 Python，想最简单 → **uv**
- 接手别人的项目 → 项目用什么你就用什么
- 新项目从零开始 → **uv**

## REPL 交互式环境

终端输入 `python` 进入，输入代码立即执行。适合快速测试。

```python
>>> 2 + 3
5
>>> type(42)
<class 'int'>
>>> _ * 2        # _ 保存上一次结果
60
>>> exit()       # 退出
```

增强版：`pip install ipython`，提供语法高亮、Tab 补全、`%timeit` 测速等。
