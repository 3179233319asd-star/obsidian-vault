# 条件判断

条件判断让程序从"一条路走到黑"变成**"能思考、会拐弯"**。

生活中处处是条件判断：
- 如果天气好 → 出门跑步
- 否则如果下小雨 → 带伞出门
- 否则 → 宅家看剧

Java / JavaScript 的同学注意：Python 不用括号括条件，不用花括号表示代码块，**用缩进代替！**

---

## 一、if 基础

```python
age = 20

if age >= 18:
    print("你已经成年了！")

print("这行不管条件如何都会执行")
```

**关键要点：**
- `if` 后面是条件，然后写一个冒号 `:`
- 冒号下面**缩进 4 个空格**的代码，就是"条件满足时要执行的内容"
- 不缩进的代码不属于 if，无论条件如何都会执行

### 为什么用缩进而不是花括号？

大多数语言用 `{}` 表示代码块：

```javascript
// JavaScript
if (age >= 18) {
    console.log("成年");
}
```

Python 的哲学是"代码怎么读就怎么写"。缩进在别的语言里只是美观问题，在 Python 里**缩进是语法要求**——不缩进或不统一就报错。

---

## 二、if-else —— 二选一

```python
temperature = 35

if temperature > 30:
    print("太热了，开空调！")
else:
    print("还行，自然风就够了。")
```

条件成立走 if 分支，不成立走 else 分支。二选一，必走一个。

---

## 三、if-elif-else —— 多分支

`elif` 是 Python 独有的写法（别的语言叫 `else if`）。

```python
score = 85

if score >= 90:
    grade = "A（优秀）"
elif score >= 80:
    grade = "B（良好）"
elif score >= 70:
    grade = "C（中等）"
elif score >= 60:
    grade = "D（及格）"
else:
    grade = "F（不及格）"

print(f"分数：{score}，等级：{grade}")
```

**执行逻辑：从上往下依次判断，第一个为 True 的分支执行后，后面全部跳过。**

### ⚠️ 常见错误

```python
# ❌ 写成 else if（这在 Python 中是嵌套 if，不是 elif！）
if score >= 90:
    grade = "A"
else if score >= 80:       # 错误！
    grade = "B"

# ✅ 正确写法
if score >= 90:
    grade = "A"
elif score >= 80:           # 正确！
    grade = "B"
```

## 四、真值测试——if 后面不一定是布尔值

Python 的 if 后面可以放任何值，会自动转为布尔。以下值被视为 **False**：

`False`, `0`, `0.0`, `""`, `[]`, `{}`, `set()`, `None`

**其他一切都是 True**（包括负数！）。

```python
# ✅ Pythonic 写法：直接把变量放在 if 后面
name = ""
if not name:                  # 空字符串是 False
    print("名字是空的")

items = []
if not items:                 # 空列表是 False
    print("列表是空的")

# ❌ 啰嗦写法（虽然也能工作）
if name != "":
if len(items) > 0:
```

## 五、比较运算符在条件中

```python
x = 15
x > 10       # True
x == 15      # True（注意是两个等号）
x != 10      # True

# 链式比较（Python 独有！）
10 < x < 20  # True — 等价于 x > 10 and x < 20
18 <= age <= 60
```

## 六、逻辑运算符组合条件

```python
age = 25
has_id = True

# and：两个都满足
if age >= 18 and has_id:
    print("可以进入")

# or：满足一个就行
if is_vip or is_employee:
    print("免票")

# not：取反
if not is_raining:
    print("出门吧")
```

## 七、in 成员判断

在 if 里用得特别多，判断某个东西是否在容器里。

```python
fruits = ["苹果", "香蕉", "橘子"]

if "香蕉" in fruits:
    print("香蕉有货！")

if "西瓜" not in fruits:
    print("西瓜没货")

# 字符串也可以用
email = "test@example.com"
if "@" in email:
    print("看起来是个邮箱地址")
```

## 八、三元表达式——if-else 的一行版

当你只是为了给变量赋值时，三元表达式更简洁。

```python
# 传统写法（4行）
if age >= 18:
    status = "成年"
else:
    status = "未成年"

# 三元表达式（1行）
status = "成年" if age >= 18 else "未成年"
```

语法：`值1 if 条件 else 值2`。注意 Python 的顺序和其他语言相反（Python 把值放前面，条件放中间）。

### 更多用法

```python
# 直接用在 f-string 里
print(f"成绩：{'及格' if score >= 60 else '不及格'}")

# 嵌套（不推荐超过两层）
grade = "A" if s >= 90 else "B" if s >= 80 else "C" if s >= 70 else "D"
```

### 另一个技巧：or 短路给默认值

```python
name = user_input or "匿名"
# 等价于：name = user_input if user_input else "匿名"
```

## 九、match-case（Python 3.10+）

`match-case` 比 if-elif 更适合做"值的匹配"。

### 基本用法

```python
match status_code:
    case 200:
        print("成功")
    case 301 | 302:          # | 表示多个值
        print("重定向")
    case 404:
        print("未找到")
    case _:                  # _ 是默认分支，类似 else
        print("未知状态")

# ⚠️ 不需要 break！每个 case 执行完自动结束
# 这和 Java/JS 的 switch 不同，不会"穿透"
```

### 结构匹配（比 switch 强大得多）

```python
# 匹配命令结构
match command.split():
    case ["quit"]:
        print("退出程序")
    case ["hello", name]:              # name 捕获值
        print(f"你好，{name}！")
    case ["move", x, y]:
        print(f"移动到 ({x}, {y})")
    case [cmd, *args]:                 # *args 捕获剩余
        print(f"未知命令：{cmd}")

# 匹配字典结构
match event:
    case {"type": "click", "x": x, "y": y}:
        print(f"点击位置 ({x}, {y})")
    case {"type": "keypress", "key": key}:
        print(f"按下按键 {key}")

# 带条件的匹配
match value:
    case int(n) if n > 0:
        print(f"正整数：{n}")
    case int(n):
        print(f"非正整数：{n}")
```

## 十、嵌套 if —— 能少用就少用

嵌套太深会让代码变成"箭头型"，很难读。

```python
# ❌ 箭头型代码
if age >= 18:
    if has_ticket:
        if is_vip:
            print("VIP 通道入场")
        else:
            print("普通通道入场")
    else:
        print("请先购票")
else:
    print("未成年人禁止入场")

# ✅ 扁平化改写
if age < 18:
    print("未成年人禁止入场")
elif not has_ticket:
    print("请先购票")
elif is_vip:
    print("VIP 通道入场")
else:
    print("普通通道入场")
```

## 十一、新手最容易踩的坑

### 坑1：把 `elif` 写成 `else if`

从 Java/JS 转过来的同学最容易犯。

```python
# ❌ Python 不认 else if
if score >= 90:
    grade = "A"
else if score >= 80:    # 报错！
    grade = "B"

# ✅ Python 用 elif
if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
```

### 坑2：缩进不一致——混用 Tab 和空格

```python
# ❌ 第一行用4个空格，第二行用Tab
if age >= 18:
    print("成年")       # 空格缩进
		print("可以投票")   # Tab缩进 → 混用报错！

# ✅ 统一用4个空格（在IDE里设置"Tab转为空格"）
if age >= 18:
    print("成年")
    print("可以投票")
```

### 坑3：忘记写冒号

```python
# ❌ 冒号忘了
# if score >= 60
#     print("及格")

# ✅ 冒号必须加
if score >= 60:
    print("及格")
```

### 坑4：if 条件里写了赋值而不是比较

```python
score = 85

# ❌ 本意是比较，但写成了赋值
# if score = 90:       # 报错！

# ✅ 比较用 ==
if score == 90:
    print("刚好90分")
```

### 坑5：以为空列表/空字符串是 True

```python
items = []
# ❌ 错误理解：有变量就是 True
# 实际上：空列表是 False！

if not items:                  # ✅ 正确的判断方式
    print("列表是空的")

# Python 的真值规则：0、空字符串、空列表、None → False
```

### 坑6：match-case 忘了不需要 break

```python
match status:
    case 200:
        print("OK")
        # ✅ 不需要 break！Python 自动不穿透
    case 404:
        print("Not Found")
    case _:
        print("Other")
# 和 Java/JS 的 switch 不同，不用担心忘记 break
```
