# 模块与包

把代码拆成多个文件，各管各的，用的时候 `import` 进来——这就是模块化的核心思想。

| 概念 | 对应 | 说明 |
|------|------|------|
| 模块 | 一个 `.py` 文件 | `utils.py` 就是一个模块 |
| 包 | 一个包含 `__init__.py` 的文件夹 | 里面可以放多个模块 |

---

## 一、为什么要用模块？

没有模块的话，你所有的代码都堆在一个 `.py` 文件里，几千行——改起 bug 来像大海捞针。

有了模块：
- 代码复用：一个工具函数写一次，到处 `import`
- 结构清晰：按功能拆分文件
- 命名空间：`math.sqrt()` 和 `your_module.sqrt()` 互不冲突

---

## 二、import 的四种方式

### 方式 1：导入整个模块（最清晰）

```python
import math

print(math.pi)          # 3.14159...
print(math.sqrt(16))    # 4.0
print(math.ceil(3.2))   # 4（向上取整）
print(math.floor(3.8))  # 3（向下取整）
```

优点：一眼看出函数来自哪个模块。
缺点：每次都要写 `math.xxx`。

### 方式 2：从模块中导入特定名字（最常用）

```python
from math import sqrt, pi, ceil

print(pi)               # 直接用，不需要 math. 前缀
print(sqrt(25))         # 5.0
```

### 方式 3：起别名

```python
import numpy as np                      # 模块别名（行业惯例）
from math import factorial as fac       # 函数别名

# 常见约定别名
# import numpy as np
# import pandas as pd
# import matplotlib.pyplot as plt

print(f"10! = {fac(10)}")   # 3628800
```

### 方式 4：星号导入（不推荐！）

```python
from random import *     # 把模块里所有公开名字全部引入
# ⚠️ 不知道引入了哪些名字，可能静默覆盖你的变量
# 只在交互式终端里快速实验时可以用
```

## 三、自定义模块

任何 `.py` 文件都是模块：

```
my_project/
  main.py        ← 主程序
  utils.py       ← 自定义模块
```

```python
# utils.py
PI = 3.14159

def add(a, b):
    return a + b

# main.py
import utils
print(utils.PI)         # 3.14159
print(utils.add(1, 2))  # 3
```

### 模块的特殊属性

```python
import utils

print(utils.__name__)    # "utils" — 模块名
print(utils.__file__)    # 文件路径
print(utils.__doc__)     # 模块顶部的文档字符串

# dir() 查看模块所有名字
public_names = [n for n in dir(utils) if not n.startswith('_')]
```

### 模块缓存机制

**模块只在首次导入时执行一次**，之后走缓存（`sys.modules`）：

```python
import utils     # 第一次：执行模块代码
import utils     # 第二次：直接从缓存取，不重新执行

import sys
print('utils' in sys.modules)   # True

# 调试时强制重新加载（一般不推荐）
from importlib import reload
reload(utils)   # 重新执行模块代码
```

### `_` 前缀——私有约定

Python **没有** `private` 关键字，但有社区约定：

```python
# utils.py
def _internal_helper():   # 单下划线开头 = "内部使用"
    return "别在外面调用我"

def public_api():         # 公开 API
    return "放心用"
```

```python
# 仍然可以访问，但"不建议"
from utils import _internal_helper   # 语法上可以

# 但 from utils import * 不会导入 _ 开头的名字！
```

## 四、`__name__` 和 `"__main__"`

让一个文件既可以当模块被导入，也可以单独运行。

```python
# utils.py
def add(a, b):
    return a + b

# 下面的代码只在"直接运行"时执行
# 如果被其他文件 import，这部分不会执行
if __name__ == "__main__":
    print("测试 add 函数：")
    print(add(1, 2))
```

原理：

| 场景 | `__name__` 的值 |
|------|-----------------|
| `python utils.py`（直接运行） | `"__main__"` |
| `import utils`（被别的文件导入） | `"utils"` |

所以 `if __name__ == "__main__":` 的意思是"这个文件是被直接运行的吗？"

### `python -m` 运行模块

```bash
python demo_main.py     → 直接运行文件
python -m demo_main     → 以模块方式运行

# 两者都会让 __name__ == "__main__"
# 常用例子：
python -m http.server 8080      # 启动简易 HTTP 服务器
python -m json.tool data.json   # 格式化 JSON
python -m pip install xxx       # 用 pip 安装包
```

## 五、包 —— 模块的文件夹

当模块多起来时（十几个、几十个），用包来组织。

```
mypackage/
  __init__.py       ← 标识"这是个包"（可以为空，但必须存在）
  module_a.py
  module_b.py
  subpackage/       ← 包里还可以有子包
    __init__.py
    module_c.py
```

```python
# 导入包里的模块
import mypackage.module_a
from mypackage import module_b
from mypackage.module_a import some_function
```

### `__init__.py` 的作用

```python
# mypackage/__init__.py
from .module_a import ClassA        # 从包内模块导入，暴露到包层面
from .module_b import ClassB

__all__ = ["ClassA", "ClassB"]      # 控制 from mypackage import * 导出什么
```

这样用户可以直接从包名导入：

```python
from mypackage import ClassA    # 不需要知道 ClassA 在 module_a 里
```

## 六、相对导入（包内部使用）

在包的**内部**，用 `.` 和 `..` 引用同包的其他模块：

```python
# subpackage/module_c.py
from . import sibling              # 同目录的模块
from .sibling import some_func     # 同目录模块的某个函数
from .. import parent_module       # 上级包的模块
from ..other_pkg import something  # 上级的另一子包
```

规则：一个 `.` 是当前目录，两个 `..` 是上级目录。

> ⚠️ 相对导入只在包内部有效，顶层脚本不能用。

## 七、常用标准库速查

Python 自带了大量"标准库"，不用安装，直接 `import`。

| 模块 | 干什么用 | 常用函数 |
|------|---------|---------|
| `math` | 数学函数 | `sqrt`, `sin`, `cos`, `ceil`, `floor`, `pi`, `e` |
| `random` | 随机数 | `randint(a,b)`, `choice(seq)`, `shuffle(seq)`, `random()` |
| `os` | 操作系统交互 | `getcwd()`, `listdir()`, `path.join()`, `environ` |
| `sys` | 系统参数 | `argv`, `exit()`, `path`, `version`, `platform` |
| `json` | JSON 处理 | `dumps()`, `loads()`, `dump()`, `load()` |
| `datetime` | 日期时间 | `datetime.now()`, `timedelta(days=7)` |
| `re` | 正则表达式 | `match()`, `search()`, `findall()`, `sub()` |
| `collections` | 高级容器 | `Counter`, `defaultdict`, `deque`, `namedtuple` |
| `itertools` | 迭代工具 | `permutations`, `combinations`, `chain`, `groupby` |
| `functools` | 高阶函数工具 | `lru_cache`, `reduce`, `wraps`, `partial` |

## 八、第三方库管理

### pip（传统方式）

```bash
pip install requests            # 安装
pip install requests==2.31.0    # 指定版本
pip uninstall requests          # 卸载
pip list                        # 看装了什么
pip freeze > requirements.txt   # 导出所有依赖
pip install -r requirements.txt # 批量安装
```

`requirements.txt` 长这样：
```
requests==2.31.0
numpy>=1.26.0
flask~=3.0.0
```

### uv（新一代，推荐）

```bash
uv init my_project                  # 创建项目
uv add requests flask               # 添加依赖
uv add --dev pytest ruff            # 开发依赖
uv remove requests                  # 移除
uv sync                             # 同步环境
uv run python main.py               # 在虚拟环境中运行
```

### 模块搜索顺序（面试常考）

```python
import sys
print(sys.path)
# 搜索顺序：
# 1. 当前脚本所在目录
# 2. PYTHONPATH 环境变量
# 3. 标准库目录
# 4. 第三方库目录（site-packages）

# ⚠️ 不要把你的文件命名为 math.py / random.py / json.py！
# 否则会覆盖标准库模块！
```

## 九、实战：自己组织一个小项目

```
my_app/
  main.py              ← 入口文件
  utils.py             ← 工具函数
  models/              ← 数据模型包
    __init__.py
    user.py
    product.py
  pyproject.toml       ← 依赖配置
```

```python
# models/__init__.py
from .user import User
from .product import Product

# main.py
from models import User, Product
from utils import format_price
```

### 模块编写最佳实践

```python
"""
✅ 推荐：
1. 模块文件名：snake_case，如 my_utils.py
2. 模块顶部写文档字符串
3. 先写 import，再写常量，再写函数/类
4. 测试代码放 if __name__ == "__main__": 里
5. 相关功能放同一模块，保持"高内聚"

❌ 避免：
1. 不要和标准库重名
2. 不要在模块顶层放大量执行代码
3. 不要循环导入（A import B，B 又 import A）
"""
```

## 新手最容易踩的坑

### 坑1：文件名和标准库/第三方库重名

```python
# ❌ 如果你写了一个 math.py，import math 会导入你自己的文件！
# 常见重名陷阱：math.py, random.py, json.py, test.py
```

### 坑2：循环导入

```python
# a.py → from b import func_b
# b.py → from a import func_a
# ❌ 循环导入，报错！

# ✅ 解决：抽共用部分到第三个文件 / 放到函数内部延迟导入
```

### 坑3：顶层脚本用相对导入

```python
# main.py（顶层入口文件）
# from .utils import add     # ❌ 顶层脚本不能用相对导入
from utils import add        # ✅ 用绝对导入
```

### 坑4：`from xxx import *` 污染命名空间

### 坑5：忘记 `if __name__ == "__main__"` 导致被导入时执行测试代码

```python
# utils.py
print("这段代码被导入了")   # 别人 import utils 时也会执行！
# ✅ 放在 if __name__ == "__main__": 里面
```

### 坑6：pip 装了包但 import 找不到

```bash
# 排查：检查 pip 对应哪个 Python / 虚拟环境是否激活
pip list
python -c "import sys; print(sys.path)"
```
