# 推导式

Python 最具代表性的语法——一行完成"循环 + 条件 + 创建容器"。

## 一、列表推导式

```python
# 基本：[表达式 for 变量 in 可迭代对象]
[x**2 for x in range(5)]               # [0, 1, 4, 9, 16]

# 带条件过滤
[x for x in range(10) if x % 2 == 0]   # [0, 2, 4, 6, 8]

# 带表达式变换
[w.upper() for w in ["hi","hello"]]     # ["HI", "HELLO"]
[c * 9/5 + 32 for c in [0, 10, 20]]    # 温度转换

# if-else（注意位置！if 在前面是变换，在后面是过滤）
["偶数" if x%2==0 else "奇数" for x in range(1, 8)]
# if 后面 → 过滤：[x for x in data if 条件]
# if-else 前面 → 变换：[A if 条件 else B for x in data]
```

### 传统 vs 推导式

```python
# 传统写法（5行）
result = []
for x in data:
    if x > 0:
        result.append(x * 2)

# 推导式（1行）
result = [x * 2 for x in data if x > 0]

# JS: data.filter(x => x > 0).map(x => x * 2)
# Java: data.stream().filter(x -> x > 0).map(x -> x * 2).collect(...)
```

### 嵌套推导式

```python
# 展平二维列表
matrix = [[1, 2, 3], [4, 5, 6], [7, 8, 9]]
flat = [num for row in matrix for num in row]
# [1, 2, 3, 4, 5, 6, 7, 8, 9]

# 等价于
flat = []
for row in matrix:
    for num in row:
        flat.append(num)

# 乘法组合
[f"{i}×{j}={i*j}" for i in range(1, 4) for j in range(1, 4)]
```

## 二、字典推导式

```python
# {键表达式: 值表达式 for 变量 in 可迭代对象}
{x: x**2 for x in range(5)}
# {0: 0, 1: 1, 2: 4, 3: 9, 4: 16}

# 带条件过滤
prices = {"苹果": 5, "车厘子": 50, "香蕉": 3, "榴莲": 80}
{p: k for k, p in prices.items() if p <= 10}
# {"苹果": 5, "香蕉": 3}

# 键值互换
{v: k for k, v in original.items()}

# 从两个列表创建
{k: v for k, v in zip(keys, values)}
# 更简单：dict(zip(keys, values))
```

## 三、集合推导式

```python
# {表达式 for 变量 in 可迭代对象}
{x**2 for x in range(5)}                 # {0, 1, 4, 9, 16}

# 不重复首字母
{name[0] for name in ["Alice", "Bob", "Anna", "Charlie"]}
# {"A", "B", "C"}

# 句子中单词长度集合
{len(w) for w in "the quick brown fox".split()}  # {3, 5}
```

## 四、生成器表达式（懒计算）

用圆括号替代方括号→按需生成，不占内存。

```python
# 列表推导式：立即生成，占内存
sum([x**2 for x in range(1000000)])    # 先创建百万个数的列表

# 生成器表达式：边算边用，不占额外内存
sum(x**2 for x in range(1000000))      # 函数只用一对括号时可省略多余括号

# 内存对比
import sys
list_comp = [x for x in range(10000)]
gen_exp = (x for x in range(10000))
sys.getsizeof(list_comp)   # 远大于
sys.getsizeof(gen_exp)     # 生成器固定大小
```

## 五、使用建议

| ✅ 该用 | ❌ 不该用 |
|---------|----------|
| 简单映射/过滤 | 逻辑复杂（难读） |
| 一行能说清楚 | 嵌套超过 2 层 |
| 生成新容器 | 循环体有副作用（打印、写文件） |

> 推导式应该让代码更清晰，不是更炫技。

## 六、新手最容易踩的坑

### 坑1：if 在前面和后面位置搞反

```python
# if 在后面 → 过滤
[x for x in range(10) if x % 2 == 0]      # [0, 2, 4, 6, 8]

# if-else 在前面 → 变换（注意位置！）
["偶数" if x % 2 == 0 else "奇数" for x in range(5)]
# ["偶数", "奇数", "偶数", "奇数", "偶数"]

# ❌ 常见错误：把 if-else 放到后面
# [x for x in range(10) if x % 2 == 0 else "奇数"]  # SyntaxError!
```

### 坑2：推导式太复杂不如写循环

```python
# ❌ 嵌套超过两层，读不懂
[x for row in matrix for x in row if x > 0 if x % 2 == 0]

# ✅ 拆成普通循环更清晰
result = []
for row in matrix:
    for x in row:
        if x > 0 and x % 2 == 0:
            result.append(x)
```

### 坑3：生成器用完就没了

```python
gen = (x**2 for x in range(3))

list(gen)   # [0, 1, 4]
list(gen)   # [] ← 空了！生成器只能迭代一次

# 如果需要反复用，改成列表
lst = [x**2 for x in range(3)]
list(lst)   # [0, 1, 4]
list(lst)   # [0, 1, 4]
```

### 坑4：`set()` 和 `{}` 搞混

```python
# {} 不一定是集合，空大括号是字典
type({})             # <class 'dict'>  ← 惊喜！
type({1, 2, 3})      # <class 'set'>

# 唯一创建空集合的方式
empty = set()
```

### 坑5：字典推导式 key 重复，后面的覆盖前面的

```python
# key 相同会被覆盖
{x: x**2 for x in [1, 1, 2, 2, 3]}
# {1: 1, 2: 4, 3: 9} ← 重复的 1 和 2 只保留最后一个
```
