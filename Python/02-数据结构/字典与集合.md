# 字典与集合

## 第一部分：字典 dict

字典是**键值对**的集合。通过"键"快速找到"值"。

生活类比：
- 一本真正的字典 → 通过"词条"（键）查"释义"（值）
- 手机通讯录 → 通过"姓名"（键）查"电话号码"（值）

Java 工程师：字典 ≈ `HashMap` / `LinkedHashMap`（Python 3.7+ 保证插入顺序）
JavaScript 工程师：字典 ≈ `Object` 或 `Map`

### 一、创建字典

```python
# 方式1：花括号（最常用）
person = {"name": "小明", "age": 18, "city": "北京"}

# 方式2：dict() 构造函数
person2 = dict(name="小红", age=20, city="上海")

# 方式3：从键值对列表创建
pairs = [("a", 1), ("b", 2)]
d = dict(pairs)             # {"a": 1, "b": 2}

# 方式4：给一批键设相同默认值
keys = ["math", "english", "python"]
scores = dict.fromkeys(keys, 0)
# {"math": 0, "english": 0, "python": 0}

# 空字典
empty = {}                  # ⚠️ {} 是空字典，不是集合
```

**键的要求：必须是不可变类型**（str, int, float, tuple, frozenset 可以；list, dict, set 不行）。

```python
# ✅ 合法的键
{(1, 2): "坐标", 42: "数字", "name": "字符串"}

# ❌ 不合法的键
# {[1, 2]: "列表不行"}   # 列表可变，不能哈希
```

### 二、取值

```python
student = {"name": "小明", "age": 18, "score": 95}

# 方式1：方括号（键不存在 → 报错 KeyError）
student["name"]          # "小明"
# student["gender"]      # ❌ KeyError

# 方式2：get()（推荐！键不存在 → 返回 None 或默认值）
student.get("name")      # "小明"
student.get("gender")    # None（不报错）
student.get("gender", "未知")  # "未知"

# 判断键是否存在
"name" in student        # True
"gender" not in student  # True
```

### 三、增 / 改

增和改的语法完全一样——**键不存在就添加，键存在就覆盖**。

```python
info = {"name": "小明", "age": 18}

info["age"] = 19              # 修改（age 已存在）
info["gender"] = "男"         # 添加（gender 不存在）
info["score"] = 95            # 添加

# update()：批量增/改
info.update({"age": 20, "city": "北京", "hobby": "编程"})
# age 被覆盖为 20，city 和 hobby 被添加

# setdefault()：键不存在才设置（不影响已有的）
info.setdefault("name", "匿名")    # name 存在，不动 → "小明"
info.setdefault("phone", "无")     # phone 不存在，添加 → "无"
```

### 四、删除

```python
data = {"a": 1, "b": 2, "c": 3, "d": 4}

val = data.pop("b")              # 删除 "b" 并返回 2
data.pop("z", "不存在")           # 键不存在返回默认值，不报错

del data["c"]                     # 直接删除
last = data.popitem()             # 删除最后一个键值对（3.7+有序）
data.clear()                      # 清空
```

### 五、遍历

```python
scores = {"数学": 95, "英语": 88, "Python": 100}

# 遍历键
for key in scores:
    print(key)                   # 等同于 for key in scores.keys():

# 遍历值
for value in scores.values():
    print(value)

# 遍历键值对（最常用！）
for subject, score in scores.items():
    print(f"{subject}: {score}分")

# 带序号遍历
for i, (subject, score) in enumerate(scores.items(), 1):
    print(f"{i}. {subject}: {score}分")
```

### 六、合并字典

```python
a = {"x": 1, "y": 2}
b = {"y": 20, "z": 30}

# 方式1：update()（原地修改）
merged = a.copy()
merged.update(b)          # {"x": 1, "y": 20, "z": 30}

# 方式2：** 解包（推荐！3.5+）
merged = {**a, **b}       # 后面的覆盖前面的

# 方式3：| 运算符（最简洁！3.9+）
merged = a | b
a |= b                    # 原地合并

# 多字典合并
{**base, **override, **extra}  # 后面覆盖前面
```

### 七、字典与 JSON

Python 字典和 JSON 结构几乎一一对应。

```python
import json

# 字典 → JSON 字符串
user = {"id": 1, "name": "张三", "skills": ["Python", "JS"]}
json_str = json.dumps(user, ensure_ascii=False, indent=2)

# JSON 字符串 → 字典
text = '{"name": "李四", "age": 30}'
parsed = json.loads(text)
print(parsed["name"])    # "李四"
```

## 第二部分：集合 set

集合有两个核心特征：
1. **自动去重**：重复元素只保留一个
2. **无序**：没有索引，不能 `set[0]`

生活类比：一个袋子，往里面放球，颜色重复的只留一个。

### 一、创建集合

```python
fruits = {"苹果", "香蕉", "橘子", "苹果"}  # 自动去重
nums = set([1, 2, 3, 2, 1, 4])          # {1, 2, 3, 4}
chars = set("Hello")                     # {"H", "e", "l", "o"}

# ⚠️ 空集合必须用 set()！
empty_set = set()
empty_dict = {}     # ⚠️ 这是空字典，不是空集合！
```

### 二、增删

```python
s = {1, 2, 3}

s.add(4)            # 添加（重复了也不报错，只是不生效）
s.update([5, 6, 7]) # 批量添加

s.remove(7)         # 删除（不存在 → KeyError）
s.discard(6)        # 删除（不存在也不报错，推荐！）
s.pop()             # 随机删除一个并返回
s.clear()           # 清空
```

### 三、集合运算（集合最强大的功能）

```python
python_组 = {"小明", "小红", "小刚", "小美", "小李"}
java_组 = {"小红", "小刚", "小张", "小王"}

python_组 & java_组     # {"小红", "小刚"} — 交集（两种都喜欢）
python_组 | java_组     # 并集（喜欢任意一种的）
python_组 - java_组     # {"小明", "小美", "小李"} — 差集（只喜欢 Python）
python_组 ^ java_组     # 对称差集（只喜欢一种的）
```

### 四、集合判断

```python
a = {1, 2, 3}
b = {1, 2, 3, 4, 5}

a <= b    # True — a 是 b 的子集
a < b     # True — a 是 b 的真子集
b >= a    # True — b 是 a 的超集
a == {3, 2, 1}     # True — 集合只看元素，不看顺序
a.isdisjoint({10, 20})  # True — 没有交集
```

### 五、实际应用

```python
# 1. 列表去重
raw = [1, 3, 2, 1, 4, 3, 2, 5]
unique = list(set(raw))           # 去重，但顺序可能变
unique_ordered = list(dict.fromkeys(raw))  # 去重且保持顺序

# 2. 快速成员判断（O(1)，比列表的 O(n) 快）
valid = {"rock", "paper", "scissors"}
if user_input in valid: ...

# 3. 权限对比
old = {"read", "write", "delete"}
new = {"read", "write", "execute", "admin"}
added = new - old       # {"execute", "admin"}
removed = old - new     # {"delete"}
```

### 六、frozenset —— 不可变的集合

```python
fs = frozenset([1, 2, 3])
# fs.add(4)   # ❌ 不能改

# frozenset 可以做字典的键（普通 set 不行）
groups = {frozenset({"小明", "小红"}): "A 组"}
```

## 第三部分：容器选择指南

| 需求 | 选哪个 | 原因 |
|------|--------|------|
| 有序、需要增删改 | **list** | 可变、有索引 |
| 有序、不可变 | **tuple** | 安全、省内存、能做 dict 键 |
| 按键查找 | **dict** | 键→值映射，O(1) 查找 |
| 去重 | **set** | 自动去重 |
| 集合运算 | **set** | 交集、并集、差集 |
| 不可变集合、做 dict 键 | **frozenset** | 可哈希 |

决策流程：
1. 需要键值对？ → dict
2. 需要去重？ → set
3. 数据需要改？ → list
4. 数据不需要改？ → tuple

## 新手最容易踩的坑

### 坑1：访问不存在的键用 `[]` 而不是 `get()`

```python
d = {"name": "小明", "age": 18}

# ❌ 键不存在直接报错
d["gender"]             # KeyError! 程序崩溃

# ✅ 用 get() 安全取值
d.get("gender")         # None — 不报错
d.get("gender", "未知")  # "未知" — 可设默认值
```

### 坑2：`{}` 是空字典，不是空集合

```python
empty_dict = {}          # 这是字典！
empty_set = set()        # 这才是空集合

type({})       # <class 'dict'>
type(set())    # <class 'set'>
```

### 坑3：把可变对象当字典键

```python
# ❌ 列表是可变的，不能当键
# d = {[1, 2]: "坐标"}     # TypeError

# ✅ 用元组（不可变）
d = {(1, 2): "坐标"}
```

### 坑4：遍历字典时修改字典

```python
d = {"a": 1, "b": 2, "c": 3}

# ❌ 遍历时改大小 → RuntimeError
for key in d:
    if d[key] < 2:
        del d[key]

# ✅ 先收集要删的键，再删除
to_delete = [k for k, v in d.items() if v < 2]
for k in to_delete:
    del d[k]
```

### 坑5：`remove` 和 `discard` 的区别

```python
s = {1, 2, 3}

s.discard(99)    # ✅ 不报错（推荐）
# s.remove(99)   # ❌ KeyError! 不存在就报错
```

### 坑6：以为集合有顺序

```python
s = {3, 1, 4, 1, 5}
print(s)          # {1, 3, 4, 5} — 顺序跟你放进去的可能不一样！
# s[0]            # ❌ 集合不支持索引

# 如果需要有序+去重 → 用 dict.fromkeys 或手动处理
```

### 坑7：`in` 查字典默认查键，不查值

```python
d = {"name": "小明", "age": 18}

"name" in d            # True — 查的是键
"小明" in d            # False — 虽然在值里，但 in 不查值
"小明" in d.values()   # True — 明确说查值才行
```

### 坑8：字典赋值的新手错误

```python
d = {"a": 1}

# ❌ 以为这样会添加，但实际上 a 被覆盖了
d["a"] = 100           # {"a": 100} — 覆盖了原来的 1

# 键存在就覆盖，键不存在才添加——增和改的语法是一样的
```
