# 四、预训练方法

## 【18】自监督学习

### 用人话解释

传统的机器学习需要人给每个数据打标签——"这张图是猫""这段话情感是正面"。但互联网上有近乎无限的文本，如果每条都要人标注，成本是天文数字。

自监督学习的鬼才思路：**让数据自己当自己的老师**。具体到 LLM——给你一段话「今天天气很___」，正确答案就是原文的下一个词「好」。不需要任何人标注，每一段文本的每一个位置都能自动变成一个"预测题"。模型就是在做无穷无尽的"完形填空"中学会了语言规律。

> 你可以这样记：自监督 = 自己出题自己批改。把文本的一部分藏起来让模型猜，原文就是标准答案。这就是为什么能够"无标注海量训练"。

### 面试怎么说

**Q：和有监督/无监督的区别？**

A：有监督 = 人工标签，无监督 = 没标签自己找规律，自监督 = 从数据自身构造标签（下个词预测）。形式上像有监督但人工零成本。

**Q：LLM 预训练为什么是自监督？**

A：任务"预测下个词"的标签就是原文本下个 token。不需要人标注。互联网所有文本自动变成带答案的练习题。

**Q：能替代人工标注吗？**

A：不能完全替代。自监督让你学会"语言什么样"，但没学会"人类想要什么样的回答"。指令遵循和安全边界仍需人工标注。

## 【19】因果语言建模（CLM）

### 用人话解释

CLM 是 LLM 的核心训练任务，一句话概括：**给你前面所有的词，猜下一个词是什么**。

比如：「法国的首都是___」，模型要预测出"巴黎"。模型只能看到"法国的首都是"，不能看到后面还没写的东西——这就是"因果"的含义。训练和推理完全一致（都是基于上文猜下文），所以训练学到的能力可以无缝迁移到推理场景。

训练效率也很高——一段"今天天气很好我出门散步"可以同时算出四个位置的预测：用"今天"猜"天气"、用"今天天气"猜"很"、用"今天天气很"猜"好"……一次前向传播并行计算所有位置的损失。

> 你可以这样记：CLM = 只看左边猜右边。GPT 的核心训练任务。好处是训练和使用的条件完全一致，不存在"训练时会但用时不会"的问题。

### 面试怎么说

**Q：CLM 和 MLM 的区别？**

A：CLM = 续写（只看左边猜右边），MLM = 完形填空（遮住中间用两边猜）。GPT 用 CLM（生成），BERT 用 MLM（理解）。

**Q：为什么生成式 LLM 选 CLM？**

A：生成天然是自回归的——只能基于已写的内容决定下一个词。CLM 的训练方式和使用方式完全一致，没有鸿沟。

**Q：能用双向信息吗？**

A：不能，只看左边。但这不是缺陷——真实生成时也只有左边可用。CLM 就是老老实实模拟真实场景。

## 【20】掩码语言建模（MLM）

### 用人话解释

MLM 是 BERT 的核心训练任务：随机把句子中 15% 的词遮住，让模型根据左右上下文猜被遮住的是什么。和 CLM 不同，MLM 能利用双向信息——被遮住的词左边和右边的上下文都能看。

为什么要只遮 15% 而不是全部？因为如果训练时全是 [MASK] 符号，模型会对 [MASK] 过度依赖，但实际用的时候（比如分类任务）输入里根本没有 [MASK]，模型就懵了。15% 是平衡点。实际还做了更精细的替换策略：选中的 15% 里，80% 真遮成 [MASK]，10% 换成随机词，10% 保留原词——让模型不完全依赖 [MASK] 符号，也要对"可能是真实词"的情况保持敏感。

> 你可以这样记：MLM = BERT 的完形填空。遮住 15% 的词，用两边信息猜。适合理解任务，不适合生成（因为生成时没有 [MASK] 也没有未来可看）。

### 面试怎么说

**Q：为什么只遮 15%？**

A：全遮 → 训练时只见过 [MASK]，微调时没有 [MASK]，输入分布不匹配。15% 是有足够训练信号又不让模型过度依赖 [MASK] 的平衡点。

**Q：MLM 怎么学到双向上下文？**

A：猜被遮住的词时，必须综合左右两侧信息。光看左边可能猜很多种，加上右边才能锁定正确答案。

**Q：为什么 GPT 不用？**

A：GPT 做生成。MLM 依赖 [MASK] 和双向上下文，生成时两个都不存在。训练和使用之间有鸿沟。
