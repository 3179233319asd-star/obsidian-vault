## 阶段一：输入表示 —— 文本怎么变成数字？

> **核心问题**：人类语言是离散符号，神经网络只能算连续向量，怎么桥接？

```
原始文本 → [分词] → Token ID 序列 → [嵌入] → 稠密向量 → [位置编码] → 带位置信息的向量 → 喂给 Transformer
```

| # | 知识点 | 角色 |
|---|--------|------|
| 1 | [[01-基础概念-Token与嵌入#【1】Token / 分词\|Token / 分词]] | 把文本切成模型认识的"单词" |
| 2 | [[01-基础概念-Token与嵌入#【2】嵌入（Embedding）\|嵌入 (Embedding)]] | 把 token 编号变成稠密向量 |
| 3 | [[01-基础概念-Token与嵌入#【3】位置编码\|位置编码]] | 注入顺序信息，弥补注意力的无序缺陷 |

**阶段一的输出 = 一个 `[序列长度 × 嵌入维度]` 的矩阵**，这是后续所有计算的起点。

> 关键认知：如果阶段一搞错了（分词不合理、嵌入维度太小、位置编码失效），后面阶段再强也白搭。这是 LLM 的"感官"。

## 阶段二：模型架构 —— 模型内部怎么"思考"？

> **核心问题**：拿到向量矩阵后，用什么结构来处理它，才能捕捉语言中的复杂关系？

```
输入矩阵
  → [多头因果注意力]   ← 每个 token 看左边所有 token，捕捉"词与词的关系"
  → [残差连接 + 层归一化]
  → [FFN]              ← 对每个 token 独立做非线性变换，储存"知识"
  → [残差连接 + 层归一化]
  → 重复 N 层           ← 浅层学语法，深层学语义和推理
  → 输出隐状态
```

**阶段二的输出 = 每个位置的隐状态向量**，最后一个位置的向量包含了整个上文的上下文信息，用来预测下一个 token。

> 关键认知：阶段二的所有组件都在同一层 Transformer 里协同工作——注意力负责"交流"（token 之间传递信息），FFN 负责"记忆"（储存学到的知识），残差+归一化负责"稳定"（让梯度能流过几十上百层）。

## 阶段三：训练体系 —— 模型怎么学会东西？

> **核心问题**：有了结构（阶段二），怎么让它从数据中学到有用的东西？

阶段三包含三个子阶段，层层递进：

### 3A - 训练引擎：底层数学

```
前向传播 → 计算损失 → 反向传播求梯度 → 优化器更新参数 → 学习率调度控制步长
```

| 20 | [[04-预训练方法#【20】掩码语言建模（MLM）\|MLM（掩码语言建模）]] | BERT 的任务：完形填空，学双向上下文 |

### 3C - 训练流程：完整的课程表

```
预训练 (海量无标注文本, CLM)
  → 让模型学会语言、知识和推理，获得"通用智能"
  ↓
SFT 有监督微调 (人工标注的指令-回答对)
  → 让模型学会遵循指令、对话
  ↓
RLHF / DPO (人类偏好数据)
  → 让模型的输出符合人类价值观（有用、诚实、无害）
```

| 24 | [[05-训练流程-三阶段#【24】RLHF（人类反馈强化学习）\|RLHF]] | 训练奖励模型 + PPO 强化学习对齐人类偏好 |
| 25 | [[05-训练流程-三阶段#【25】DPO（直接偏好优化）\|DPO]] | 跳过奖励模型，直接用偏好数据优化 |

> 关键认知：
> - **3A 是数学引擎**，不管训练什么神经网络都要用（不仅 LLM）
> - **3B 是任务定义**，决定了模型学到什么能力（CLM→生成能力，MLM→理解能力）
> - **3C 是训练管线**，从通用到专用、从能力到对齐的三步走
>
> 三者关系：3A（怎么算） + 3B（算什么） + 3C（分几步算）= 完整的训练体系

## 阶段四：推理与生成 —— 训练好了怎么用？

> **核心问题**：模型训练完后，怎么让它产出又好又多样的文本？

```
用户 Prompt
  → 模型前向传播（利用 KV Cache 避免重复计算）
  → 输出下一个 token 的概率分布
  → 采样策略（温度 + Top-p + Top-k）选出一个 token
  → 追加到输入末尾
  → 循环直到 <EOS> 或达到最大长度
```

|---|--------|---------------|

| 27 | [[06-推理与生成#【27】KV Cache\|KV Cache]] | 为什么不用每步都重算整个序列？ |
| 28 | [[06-推理与生成#【28】温度\|温度]] | 怎么控制生成的随机性？ |
| 29 | [[06-推理与生成#【29】Top-p / Top-k 采样\|Top-p / Top-k 采样]] | 怎么避免生成低质量的垃圾 token？ |
| 30 | [[06-推理与生成#【30】Beam Search\|Beam Search]] | 怎么找到整体最优的序列？（翻译等确定性任务用） |

> 关键认知：阶段四的每个知识点都是为了解决生成过程中的一个具体痛点——KV Cache 解决速度，温度/Top-p 解决多样性，Beam Search 解决最优性，Function Calling 扩展能力边界。

## 阶段五：规模化 —— 为什么大模型神奇 & 怎么让它跑起来？

> **核心问题**：当模型从小变大，会发生什么？怎么让大到离谱的模型能实际训练和部署？

阶段五由三块内容交叉构成：

### 5A - 能力现象：变大的"质变"

```
缩放法则 (#38)：参数↗ + 数据↗ + 算力↗ → 性能可预测提升（幂律关系）
    ↓
涌现能力 (#35)：超过 ~10B 参数后突然出现小模型完全没有的能力
    ├── 上下文学习 (#33)：给几个示例就学会新任务，不用改参数
    └── 思维链 (#34)：先推理再回答，多步推理准确率飙升
    ↓
但同时也带来问题：
    ├── 幻觉 (#36)：编造事实，信心满满地胡说
    └── 需要对齐 (#37)：让模型遵循人类价值观（回到阶段三的 RLHF/DPO）
```

|---|--------|------|
| 38 | [[07-重要能力与现象#【38】缩放法则\|缩放法则]] | **预测**：多大的投入换多大的收益 |
| 35 | [[07-重要能力与现象#【35】涌现能力\|涌现能力]] | **质变**：大到一定程度突然"开窍" |
| 33 | [[07-重要能力与现象#【33】上下文学习\|上下文学习]] | 涌现的典型代表：无梯度适应新任务 |
| 34 | [[07-重要能力与现象#【34】思维链\|思维链]] | 涌现的典型代表：step-by-step 推理 |
| 36 | [[07-重要能力与现象#【36】幻觉\|幻觉]] | 大模型的核心缺陷 |
| 37 | [[07-重要能力与现象#【37】对齐\|对齐]] | 用 RLHF/DPO 解决安全和价值观问题 |

## 阶段六：评估反馈 —— 怎么衡量好不好？

> **核心问题**：做完以上所有事情后，怎么知道模型到底有多好？

| # | 知识点 | 作用 |

1. **先理解阶段一→二→三→四**，这是 LLM 的"教科书流程"——输入→结构→训练→使用
2. **再看阶段五**，理解为什么大模型有"神奇"表现，以及怎么工程化地做出来
3. **最后看阶段六**，了解怎么评判好坏
