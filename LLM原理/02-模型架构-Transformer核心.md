# 二、模型架构（Transformer 核心部件）

## 【4】Transformer 整体结构

**是什么**：一个由 N 层相同 block 堆叠而成的神经网络。每层包含两个核心子层——多头自注意力（让每个 token 看其他所有 token）和前馈网络 FFN（对每个 token 独立做变换），每个子层后跟残差连接和层归一化。

**为什么这样设计**：在 Transformer 之前，处理序列的主流是 RNN/LSTM——它们必须按顺序一步步算，第 5 个 token 要等前 4 个算完。Transformer 的注意力机制让任意两个位置的 token 直接交互，路径长度都是 O(1)，所以既能捕捉长距离依赖，又能完全并行计算。残差连接 + 层归一化则解决了深层网络的训练难题，让这个架子可以堆到上百层。

**和周边的联系**：Transformer 是整个 LLM 的身体骨架。后续几乎所有知识点——注意力变体、训练优化、推理加速——都是在这个骨架上做文章。

### 面试问答

**Q：Transformer 的核心创新？**

A：完全用注意力替代 RNN 的序列依赖，任意两个 token 交互距离都是 O(1)，可并行训练。多头注意力同时捕捉多种关系。残差 + 归一化让网络能堆上百层。

**Q：为什么比 RNN 更适合并行？**

A：RNN 必须串行——算第 5 个要等前 4 个。Transformer 一次对整个序列做矩阵乘法，GPU 的几千个核心全能用上。

**Q：结构图怎么画？**

A：编码器 = 多头自注意力 → 残差+归一化 → FFN → 残差+归一化，重复 N 次。解码器中间多一层交叉注意力。

## 【5】仅解码器（Decoder-only）

**是什么**：只使用 Transformer 解码器部分的架构，用因果注意力（单向，只能看到左边的 token），自回归地逐 token 生成输出。GPT、LLaMA 等所有主流生成式 LLM 都采用这个架构。

**为什么这样设计**：文本生成天然是单向的——你写下第 10 个词时，不可能知道后面还没写的词是什么。Decoder-only 的因果掩码完美模拟了这个约束，训练时和推理时的行为完全一致（都只能看到上文）。这避免了 Encoder-Decoder 架构中训练和推理可能出现的分布不匹配问题。另外，去掉编码器和交叉注意力后架构更简单，单模型可以同时做理解和生成。

**和周边的联系**：Decoder-only 必须搭配 CLM（因果语言建模）训练目标，两者是架构和任务的天然绑定。因果注意力是 Decoder-only 的核心机制。

### 面试问答

**Q：为什么 GPT 用 Decoder-only，BERT 用 Encoder-only？**

A：任务不同。GPT 做生成，必须单向自回归；BERT 做理解，需要双向上下文。Decoder-only 天然匹配生成，Encoder-only 天然匹配理解。

**Q：Decoder-only 怎么生成文本？**

A：给 prompt → 前向传播 → 选一个 token → 追加到输入 → 再前向传播 → 选下一个 → 循环到 `<EOS>` 或最大长度。

**Q：能做理解任务吗？**

A：能，取最后一个 token 的隐状态接分类头。因果注意力下，最后一个 token 已经看过前面所有内容。

## 【6】编码器与解码器的区别

**是什么**：原版 Transformer 由编码器（Encoder）和解码器（Decoder）两部分组成。编码器用双向自注意力，每个 token 可以看到序列中所有其他 token，适合"理解"整段输入。解码器用因果自注意力，只能看左边的 token，还有一层交叉注意力以编码器输出为 K/V 来引入源语言信息，适合"生成"输出。

**为什么这样设计**：理解任务需要全局上下文——判断一句话的情感时，后面的词可能决定整句是正面还是负面。生成任务必须保持因果性——你只能基于已经写出来的内容决定下一个词。原版 Transformer 为机器翻译设计：编码器先把英文全文深度理解成向量表示，解码器基于这个理解一个词一个词地拼出中文翻译。

**和周边的联系**：编码器对应 MLM 预训练任务（BERT），解码器对应 CLM 任务（GPT），Encoder-Decoder 完整架构对应 T5/BART 等模型。

### 面试问答

**Q：Encoder 和 Decoder 注意力有什么不同？**

A：Encoder 自注意力双向（看全部），Decoder 自注意力单向 + 因果掩码（只看左边）。Decoder 多一层交叉注意力，K/V 来自 Encoder 输出。

**Q：什么场景用 Encoder-Decoder？**

A：输入和输出是不同格式或语言——翻译、摘要。Encoder 做理解，Decoder 做生成。

**Q：BERT 和 GPT 各属哪种？**

A：BERT = Encoder-only（阅读理解型），GPT = Decoder-only（写作生成型）。

## 【7】因果注意力

**是什么**：在注意力计算中加上一个上三角掩码矩阵，把"未来"位置的注意力分数设为负无穷，softmax 后这些位置的权重变为 0。效果就是每个 token 只能关注自己和左边的 token。

**为什么这样设计**：如果不加因果掩码，训练时模型会偷看未来的词作为预测依据——loss 看起来很低，但实际什么也没学到，一到推理阶段（没有未来可以偷看）就原形毕露。因果注意力保证了训练和推理的条件一致，让模型学会真正地从已知推导未知。

**和周边的联系**：因果注意力是 Decoder-only 架构的核心，也是 CLM 预训练任务能够正确运作的前提。推理时因果注意力 + KV Cache 是高效率自回归生成的关键组合。

### 面试问答

**Q：因果掩码怎么实现？**

A：算完 Q×K^T 后，把 j>i 的位置设为负无穷 → softmax 后这些位置的权重精确归零。整个过程用矩阵运算一次完成。

**Q：不加掩码会怎样？**

A：训练时偷看答案（用未来的词预测现在），loss 很低但啥也没学到。推理时未来不存在，生成一塌糊涂。

**Q：训练和推理有什么不同？**

A：训练可完全并行（已有完整序列，掩码一次性遮住）。推理必须串行（每次生成一个再算下一个），KV Cache 加速推理。

## 【8】多头注意力

**是什么**：把注意力机制复制多份，每份（称为一个"头"）有自己独立的 Q、K、V 投影矩阵。每个头从不同角度关注 token 之间的关系——比如一个头关注主谓搭配，另一个头关注代词指代。所有头的输出拼接后做一次线性变换融合。

**为什么这样设计**：单头注意力的表达能力有限——一个 token 对其他所有 token 只能产出一种"混合表示"。如果有多个头，不同的头可以专门负责不同子任务。但头数不能无限多——每个头的维度 = d_model / num_heads，头太多会导致每个头的维度太小，装不下足够信息。

**和周边的联系**：多头注意力是 Transformer 计算量最大的部分（O(L²×d)），也是 KV Cache、稀疏注意力等优化的主要目标。Q、K、V 的维度设计直接影响了 RoPE 位置编码的嵌入方式。

### 面试问答

**Q：每个头独立吗？怎么合并？**

A：完全独立，各有自己的 Q/K/V 投影。输出在特征维度拼接，再用 W_O 做线性变换融合回原始维度。

**Q：头数越多越好吗？**

A：不是。每个头维度 = d_model / heads。头太多 → 每头维度太小 → 装不下有效信息。实践每头至少 64 维，常见 8~128 个头。

**Q：为什么维度要除以头数？**

A：不除的话所有头拼起来维度 = d_model × heads，下一层没法接。除了刚好拼回 d_model。

## 【9】前馈网络（FFN）

**是什么**：每个 Transformer 层中的第二部分，一个简单的两层全连接网络——先把输入从 d_model 升维到 4×d_model，过激活函数（GELU/Swish），再降回 d_model。

**为什么这样设计**：如果把注意力比作"token 之间的交流"（交换信息、建立关系），FFN 就是每个 token 的"独立思考时间"——注意力之后已经收集了上下文信息，FFN 对每个 token 独立做非线性变换，把信息加工成知识储存起来。先升维再降维的"瓶颈结构"被实验证明非常有效——升维给变换提供了更大的空间，降维恢复到原来的尺寸以便接残差连接。FFN 的参数量大约是注意力部分的两倍，模型的大部分"知识"存在这里。

**和周边的联系**：FFN 是 Transformer 中参数量最大的部分，因此也是 MoE（混合专家）技术改造的主要目标——把单个 FFN 替换成多个专家，每次只激活少数几个。

### 面试问答

**Q：为什么先升维再降维？**

A：升维给非线性变换更大的操作空间，降维收回来接残差。实验证明比同维度变换效果好得多。

**Q：为什么用 GELU 不用 ReLU？**

A：ReLU 负数一刀切为 0，神经元容易"死亡"。GELU 平滑过渡，负值也有非零输出，梯度始终可流。深层网络里差异明显。

**Q：FFN 和注意力哪个参数量大？**

A：FFN 约是注意力的两倍（8d² vs 4d²）。模型的"知识"主要存在 FFN 里。

## 【10】残差连接

**是什么**：每个子层（注意力或 FFN）的输出不直接传给下一层，而是输出 = 子层(输入) + 输入本身。相当于给每一层修了一条"高速公路"，让信息可以不经过任何变换直接跳到后面。

**为什么这样设计**：深层网络的核心难题是梯度消失——反向传播时梯度经过几十层乘法后会衰减到几乎为零，前面的层根本学不到东西。残差连接求导时 ∂L/∂x = ∂L/∂y × (1 + ∂F/∂x)，那个 "+ 1" 保证梯度有一条始终畅通的直达通道。所以即使堆到 100 层以上，浅层的梯度依然健康，能有效学习。

**和周边的联系**：残差连接 + 层归一化是稳定训练深层 Transformer 的关键组合。Pre-norm（先归一化再子层再残差）是当前 LLM 的主流选择。

### 面试问答

**Q：残差连接怎么解决梯度消失？**

A：y = x + F(x) 求导：∂L/∂x = ∂L/∂y × (1 + ∂F/∂x)。+1 保证梯度有一条不衰减的直达通道通到浅层。

**Q：为什么残差之后还要层归一化？**

A：残差把两条路径的值直接加起来，分布可能偏移或方差变大。层归一化重新标准化，让下一层收到的输入稳定。

**Q：Pre-norm 和 Post-norm 哪个好？**

A：Pre-norm（先归一化再子层再残差）梯度更稳定，对学习率不敏感，现代 LLM 默认用。

## 【11】层归一化

**是什么**：对每个 token 的向量在特征维度上做标准化——减均值除以标准差（LayerNorm）或只除以均方根（RMSNorm），让每层输出的数值分布保持稳定。

**为什么这样设计**：神经网络训练有一个核心假设：每一层接收到的输入应该服从相似的数据分布。如果不做归一化，前一层输出的数值可能偏移或放大，导致下一层的计算变得不稳定（梯度爆炸或消失）。LayerNorm 在"每个样本内部"的特征维度上归一化，不依赖 batch 中其他样本，因此完美适配文本变长、batch 大小不固定的 NLP 场景。RMSNorm 是 LayerNorm 的简化版——省掉减均值这一步，速度更快效果持平，LLaMA 用的就是它。

**和周边的联系**：层归一化通常紧跟在残差连接之后（Pre-norm 则是先归一化再子层）。归一化位置的选择（Pre-norm vs Post-norm）直接影响训练的稳定性和最终效果。

### 面试问答

**Q：LayerNorm 和 BatchNorm 的区别？为什么 NLP 用 LayerNorm？**

A：BatchNorm 在 batch 内跨样本归一化，依赖 batch 大小且变长序列时不稳定。LayerNorm 在单样本的特征维度上归一化，跟 batch 和序列长度无关，天然适合 NLP。

**Q：RMSNorm 简化了什么？**

A：LayerNorm = 减均值 + 除以标准差。RMSNorm 发现减均值对效果影响不大，省掉后快 7%~10%。LLaMA 用的就是 RMSNorm。

## 【12】激活函数（GELU / Swish）

**是什么**：FFN 中间的非线性函数。GELU = x × Φ(x)（x 乘以标准正态累积概率），Swish = x × sigmoid(x)。两者的曲线很像——正值接近 x，负值不是 0 而是接近 0 的负数。

**为什么这样设计**：如果 FFN 的两层全连接之间没有激活函数，两层线性变换就等价于一个线性变换（两个矩阵相乘 = 一个矩阵），无论堆多少层也表达不了非线性关系。ReLU 虽然简单，但负数直接截断为 0 会导致梯度——神经元容易"死亡"。GELU/Swish 在负数区域有一条小尾巴，梯度始终能流，而且这个平滑特性在深层 Transformer 中带来了更好的优化效果。GELU 还有一个概率解释——相当于在说"这个值有多大概率是显著的"，显得更"聪明"。

**和周边的联系**：激活函数决定了 FFN 层的非线性表达能力。不同激活函数的选择（GELU vs Swish vs SiLU）在最终效果上差别不大，但 GELU 是 BERT/GPT 早期验证过的安全选择。

### 面试问答

**Q：为什么不用 ReLU？**

A：ReLU 负数一刀切为 0，神经元可能永久死亡。GELU 平滑过渡，负值有小输出，梯度不断流。深层 Transformer 里 GELU 效果明显更好。

**Q：Swish 和 GELU 的特点？**

A：Swish = x × sigmoid(x)，GELU = x × Φ(x)，形状相似——正值放行、负值抑制但不消灭。LLM 里两者差距不大，GELU 历史更久。

**Q：激活函数在 FFN 中起了什么作用？**

A：没有它，两层全连接就是线性变换，无论多少层都等于一层。激活函数引入非线性，让网络能拟合复杂的语言规律。
