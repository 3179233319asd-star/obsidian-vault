# 三、训练核心引擎（优化与更新）

## 【13】损失函数

**是什么**：一个衡量"模型预测有多不准"的公式。LLM 预训练用交叉熵损失：Loss = -Σ (真实分布 × log(预测分布))。对于每个 token 位置，模型预测正确 token 的概率越高，loss 越小。

**为什么这样设计**：训练需要明确的目标信号——模型怎么知道自己的预测是好是坏？交叉熵天然适合分类类任务：预测越准 → loss 越低 → 梯度越小（已经做得好就别乱动）；预测越离谱 → loss 暴涨 → 梯度很大（赶紧纠正）。而且交叉熵的梯度形式极其简洁：∂Loss/∂logit = y_pred - y_true，模型不用额外计算就可以知道每个候选词的梯度方向。

**和周边的联系**：损失函数的输出是反向传播的起点——损失值对每个参数求偏导，得到梯度，再传给优化器做参数更新。困惑度就是损失函数的指数变换。

### 面试问答

**Q：预训练用什么损失函数？**

A：交叉熵损失。对每个 token 位置，最大化正确 token 的预测概率。预测越准 loss 越低，越离谱 loss 越高。

**Q：为什么用交叉熵？**

A：梯度形式简洁（∂Loss/∂logit = y_pred - y_true），是凸函数优化稳定，分类任务标配。

**Q：怎么判断收敛？**

A：看验证集损失——不降了就该停了。LLM 预训练通常不会过拟合（数据太多），更多基于预算决定。

## 【14】反向传播

**是什么**：从输出层往输入层逐层计算损失函数对每个参数的偏导数（梯度）的算法。利用链式法则，把复杂的多层的导数计算拆成一系列简单的局部导数相乘。

**为什么这样设计**：一个 LLM 有几十亿个参数，手动给每个参数求导是不可能的。反向传播的美妙在于"自动化"——你把前向计算过程写成代码，框架自动记录每一步的运算（构建计算图），然后从最终损失开始，按相反顺序逐段计算每步的局部梯度再乘起来。任何可微的运算都能参与这个过程，所以网络结构可以任意复杂。

**和周边的联系**：反向传播算出的梯度交给梯度下降/AdamW 做参数更新。残差连接的存在让反向传播中的梯度有一条不衰减的"1 通道"，是保障深层 Transformer 能训练的核心设计。

### 面试问答

**Q：链式法则怎么理解？**

A：y = f(g(x)) → dy/dx = f' × g'。神经网络就是几万个基本运算复合，反向传播按相反顺序逐段求导再乘起来。

**Q：为什么说反向传播是自动求导的基础？**

A：任何可微运算的导数已知，链式法则把它们串起来。框架自动记录计算图，反向时逐段算梯度，不需要人手推导。

**Q：Transformer 里怎么通过残差连接？**

A：y = x + F(x) → ∂L/∂x = ∂L/∂y × (1 + ∂F/∂x)。1 通道让梯度直通浅层，这是残差能堆 100 层的数学原因。

## 【15】梯度下降

**是什么**：参数更新的基本算法——参数沿着梯度的反方向走一小步：θ_new = θ_old - 学习率 × 梯度。梯度指向"最陡峭的上坡方向"，反方向就是下坡方向。

**为什么这样设计**：把损失函数想象成一片山脉（参数空间），我们的目标是走到最低的谷底（损失最小的参数）。梯度告诉你当前位置往哪个方向最陡，学习率决定这一步迈多大。步太大可能从山一侧跳到另一侧来回震荡，步太小太阳下山还走不到谷底。实际训练不用全量梯度下降（算完整个数据集才更新一次），而是用 Mini-batch——每次随机抽几百条算近似梯度，又快又准。

**和周边的联系**：梯度下降给出更新公式的框架，但具体怎么算学习率、怎么加动量，由 AdamW 等优化器负责。学习率调度负责训练过程中动态调整步幅。

### 面试问答

**Q：更新公式？学习率作用？**

A：θ_new = θ_old - η × ∇L。η 决定步幅——太大震荡不收敛，太小收敛太慢。像下山：梯度说方向，学习率说步子多大。

**Q：为什么不用全量梯度下降？**

A：LLM 数据集几 TB，跑一轮要几天；所有样本梯度全存内存会爆。Mini-batch 每次随机抽几百条算近似梯度，快且等价。

**Q：怎么判断卡在局部最小值？**

A：高维空间真正局部最小值很罕见，多是鞍点。看梯度范数——损失还高但梯度很小说明可能卡住了。AdamW 的自适应学习率通常能冲出来。

## 【16】AdamW 优化器

**是什么**：梯度下降的增强版，结合了动量（累积历史梯度方向，让下山更稳）、自适应学习率（每个参数根据自己的历史梯度大小独立调整步幅）、以及解耦的权重衰减（把参数持续往 0 方向拉防止过大）。

**为什么这样设计**：裸梯度下降有三个问题——方向波动大（这步往东下一步往西）、所有参数共用一个学习率（但有些参数需要大步有些需要小步）、参数容易膨胀过大。Adam 用动量和自适应学习率解决了前两个，但权重衰减的实现有 bug（和自适应学习率耦合在一起）。AdamW 把权重衰减从梯度更新中独立出来，做了"解耦"——先梯度更新，再参数衰减，两步各干各的。这个微小的改动让收敛速度和泛化能力都大幅提升，全面取代了 Adam。

**和周边的联系**：AdamW 是训练几乎所有现代 LLM 的默认优化器。它的超参数（β1、β2、学习率）学习率调度策略有强依赖——AdamW + 余弦退火是目前的标准组合。

### 面试问答

**Q：AdamW 比 Adam 改了什么？**

A：Adam 权重衰减和梯度更新耦合在一起。AdamW 解耦——梯度更新和权重衰减分开做，收敛更快泛化更好。

**Q：为什么需要权重衰减？**

A：防止参数膨胀过大。大参数对噪声敏感易过拟合。权重衰减持续把参数往 0 方向拉，保持参数在合理范围。

**Q：超参数怎么设？**

A：β1=0.9（动量），β2=0.999（自适应），ε=1e-8（防除零），weight_decay=0.01~0.1，学习率 1e-4~6e-4。

## 【17】学习率调度

**是什么**：训练过程中动态调整学习率的策略。通常分三阶段——预热（从接近 0 线性增长到目标值）、主体训练（保持或缓慢下降）、衰减（余弦曲线平滑降到接近 0）。

**为什么这样设计**：训练刚开始时权重随机初始化，梯度方向和大小都不稳定，一上来用全速学习率可能一个大更新把模型推到错误的方向。预热让模型先用小步"探路"。训练后期快收敛时，大学习率会导致参数在最优解附近跳来跳去无法精细调节，需要衰减到很小的学习率做最后的精细收敛。余弦退火是目前最主流的衰减策略——曲线平滑自然，而且支持"重启"（跳回高学习率再降一轮），帮助模型探索更优区域。

**和周边的联系**：学习率调度的上限学习率、预热步数通常和 AdamW 配置、模型规模、batch size 一起调参。微调阶段用的学习率和调度策略一般比预训练更保守。

### 面试问答

**Q：为什么需要预热？**

A：初始权重随机，梯度方向不稳定。一上来大学习率可能把模型推到极差区域。预热让模型先小步站稳再加速。

**Q：余弦退火有什么好处？**

A：下降曲线平滑——开始慢、中间快、最后又慢。最后阶段精细收敛。支持重启跳出局部区域。

**Q：学习率设错会看到什么现象？**

A：太大 → loss 剧烈震荡甚至 NaN。太小 → loss 像蜗牛爬，训了几万步几乎没动静。
