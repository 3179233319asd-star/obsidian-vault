# 三、训练核心引擎（优化与更新）

## 【13】损失函数

### 用人话解释

模型训练需要一个"打分员"——每做一次预测，就告诉它"你差了多少分"。损失函数就是打分员。LLM 用交叉熵损失：模型预测下一个词的概率分布，然后计算正确答案对应的那个概率有多低。

直觉：正确答案是"苹果"，模型预测"苹果"概率 0.9 → -log0.9 ≈ 0.1，扣分很少，差不多对。预测概率只有 0.01 → -log0.01 = 4.6，扣分爆炸，说明错得厉害需要大改。

> 你可以这样记：损失函数 = 考试扣分。答得越准扣分越少，答得越离谱扣分越多。模型的目标就是少扣分。

### 面试怎么说

**Q：预训练用什么损失函数？**

A：交叉熵。对每个 token 位置最大化正确 token 的概率。越准 loss 越低，越离谱 loss 越高。

**Q：为什么用交叉熵？**

A：梯度形式简洁（∂Loss/∂logit = y_pred - y_true），凸函数优化稳定，分类任务标配。

**Q：怎么判断收敛？**

A：看验证集损失——不降了就该停了。LLM 预训练一般不会过拟合（数据太多），更多基于预算决定。

## 【14】反向传播

### 用人话解释

损失函数告诉你"这次考试一共扣了多少分"，接下来要做的是找出每个参数该为这个错误负多少责任——这个责任就是梯度。反向传播用链式法则从最后一层往前逐层算，把每个参数对最终损失的贡献计算出来。

一个 LLM 有几十亿参数，好在整个过程完全自动化——你写前向代码，PyTorch 等框架记录计算图，一键算出所有梯度。

> 你可以这样记：反向传播 = 考完试从头到尾复盘，每个参数都检讨"我的错在哪，我该负多少责"，从最后一步往回倒推。

### 面试怎么说

**Q：链式法则怎么理解？**

A：y = f(g(x)) → dy/dx = f' × g'。网络是几万个基本运算复合，反向从后往前逐段乘。框架全自动完成。

**Q：Transformer 里怎么通过残差连接？**

A：y = x + F(x) → ∂L/∂x = ∂L/∂y × (1 + ∂F/∂x)。"1"让梯度直通浅层，这是残差能堆 100 层的数学原因。

## 【15】梯度下降

### 用人话解释

知道了每个参数的"责任"（梯度），下一步就是让参数朝着"少犯错误"的方向挪一挪：θ_new = θ_old - 学习率 × 梯度。梯度指向"错误增加最快的上坡方向"，参数就往反方向走。

把损失函数想象成一座山，目标是走到最低的谷底。梯度告诉你脚底往哪个方向最陡（上坡），反方向就是下坡。学习率决定一步迈多大——太大从山这边跳到那边来回震荡，太小走到天黑还不到谷底。实际用的 Mini-batch——每次随机抽几百条算近似梯度，不用等全部数据过完才更新。

> 你可以这样记：蒙着眼在山上找最低点。梯度告诉"脚下往哪儿最陡"，学习率决定"步子多大"。Mini-batch 是走一小段就根据这一段的地形调整方向。

### 面试怎么说

**Q：更新公式？学习率作用？**

A：θ_new = θ_old - η × ∇L。η 控制步幅——太大震荡不收敛，太小收敛太慢。

**Q：为什么不用全量梯度下降？**

A：数据集几 TB，算一轮要几天，内存也爆。Mini-batch 每次几百条算近似梯度，快且效果等价。

**Q：怎么判断卡住了？**

A：几十亿维空间里真正局部最小值很罕见，多是鞍点。损失还高但梯度很小 → 可能卡住。AdamW 通常能冲出来。

## 【16】AdamW 优化器

### 用人话解释

裸梯度下降有三个毛病：方向波动大（这步往东下步往西）、所有参数共用一个学习率（有的该大步有的该小步但全一样）、参数容易膨胀过大。AdamW 三管齐下：

1. **动量**：不光看当前梯度方向，还累加历史方向 → 下山更稳，不会东摇西晃
2. **自适应学习率**：每个参数有自己的独立学习率，根据历史自动调 → 该大的大该小的小
3. **权重衰减**：每次更新后参数自动缩一点点 → 防止参数膨胀失控

AdamW 相比 Adam 的关键改进是"解耦"——权重衰减从梯度更新中独立出来。这个微小改动让收敛速度和最终效果大幅提升，现已全面替代 Adam。

> 你可以这样记：AdamW = 动量（跑得稳）+ 自适应学习率（步子智能）+ 独立权重衰减（别吃太胖）。LLM 训练标配。

### 面试怎么说

**Q：比 Adam 改了什么？**

A：Adam 的权重衰减和自适应学习率耦合在一起影响效果。AdamW 解耦——先梯度更新，再独立做权重衰减。收敛更快泛化更好。

**Q：为什么需要权重衰减？**

A：防止参数值变大失控。大参数对噪声敏感容易过拟合。衰减持续轻轻往回拉。

**Q：超参数？**

A：β1=0.9，β2=0.999，ε=1e-8，weight_decay≈0.01~0.1，学习率 1e-4~6e-4。

## 【17】学习率调度

### 用人话解释

从头到尾用同一个学习率不合理。刚开始训练时权重是随机的，梯度方向不稳定，大步走容易摔飞。快收敛时已经很接近最优解，大步走会来回跳无法精细调整。

所以学习率要按计划变化：**预热**阶段从接近 0 线性增加到目标值（先小步探路）→ **主体训练**保持或缓降 → **余弦退火**像余弦曲线平滑降到接近 0（最后的精调）。余弦退火是目前最主流的策略，曲线自然，还支持"重启"（跳回高值再降一轮），帮模型探索更好的区域。

> 你可以这样记：学习率调度 = 开车油门控制。刚起步轻踩预热，路况好了加速，快到了缓缓收油。余弦退火刚好符合这个节奏。

### 面试怎么说

**Q：为什么需要预热？**

A：初始权重随机，梯度方向不稳定。一上来大学习率可能把模型推到极差区域。预热先小步站稳再加速。

**Q：余弦退火的好处？**

A：平滑曲线——开头慢、中间快、最后又慢。最后阶段精细收敛。支持重启跳出局部区域。

**Q：设错会看到什么？**

A：太大 → loss 剧烈震荡甚至 NaN。太小 → 几万步 loss 几乎不动，浪费算力。
