# 九、长上下文技术

## 【43】上下文窗口

### 用人话解释

上下文窗口就是模型"一次最多能读多少 token"。受三个东西限制：注意力计算量随长度平方增长（4K→32K，计算量涨 64 倍）、位置编码只能处理训练见过的长度范围、训练数据里缺少超长文本的样本。

扩大窗口不是免费的——训练成本暴涨，而且位置编码在超出训练长度后，远近依赖的比例可能失衡（近处看得太清，远处完全看不见）。所以通常不是"训练一个超长窗口的模型"，而是"训一个中等窗口，再用几千步长文本继续训练来拓展"。

> 你可以这样记：上下文窗口 = 模型的"一次能记多少"。就像你一次能记住的聊天内容有限，要扩大要么加内存要么换记忆策略。

### 面试怎么说

**Q：受哪些限制？**

A：注意力 O(n²) 计算量、位置编码只见过短序列训练、训练数据缺长文本样本。三重约束。

**Q：扩大会怎样？**

A：计算量暴增（32K ≈ 4K 的 64 倍），位置编码可能远近失衡，训练成本翻倍。通常需继续训练适应。

**Q：百万 token 窗口怎么做到的？**

A：Ring Attention（序列分多设备）、FlashAttention（显存优化）、位置外推（RoPE 缩放）、多机共享 KV Cache。组合拳。

## 【44】位置外推

### 用人话解释

模型训练时最多见过 4096 长度的序列。但推理时用户可能扔一本小说进来（10 万 token）。怎么让位置编码支持没见过的位置？这就叫位置外推。

RoPE 天然有利——它编码的是"相对距离"而不是"绝对位置"。训练时模型已经见过 0~4096 之间所有的距离模式，推理时大部分距离（尤其是近的）在训练中都见过，只有超大距离是新的。NTK 缩放是目前最好的方法：RoPE 有多个频率（高的处理近处，低的处理远处），NTK 只调低频（让远处能看更远），不动高频（保持近处分辨率），效果最好且基本不需额外训练。

> 你可以这样记：位置外推 = 训练时最多跑步 4 公里，但用特殊技巧让你能跑 10 公里不掉速。RoPE+NTK 的技巧是：近处不变（已经跑得很好），远处调整（稍微换个频率跑）。

### 面试怎么说

**Q：为什么 RoPE 利于外推？**

A：RoPE 编码相对距离。训练已见过绝大多数短距离，只需把新的大距离映射到训练见过的范围。

**Q：线性缩放和 NTK 的区别？**

A：线性缩放所有位置索引乘一个系数，粗暴但损害短距离精度。NTK 只调低频（远距离），不动高频（近距离）。效果更好基本不需微调。

**Q：需要微调吗？**

A：线性需要（几百~几千步）。NTK 零微调也基本能用，但要最佳效果仍建议少量继续训练。

## 【45】稀疏注意力

### 用人话解释

全注意力要求每两个 token 之间都算一遍——1 万个 token 就要算 1 亿个注意力分数。大多数 token 之间其实没啥关系（第 1 个词的"我"和第 5000 个词的"了"八竿子打不着），全算一遍纯属浪费。

稀疏注意力就是只算有意义的组合：每个 token 看附近 w 个邻居（滑动窗口，保证局部连贯）、隔 d 个看一个（步长采样，提供远处概览）、几个"哨兵 token"跟所有人交互（全局节点，充当信息中转站）。单层感受野虽小，但经过 N 层堆叠后有效感受野 = N × 窗口大小，信息能在远距离间传播。

> 你可以这样记：稀疏注意力 = 不是所有人两两交流，而是只跟自己附近圈子的人聊天 + 听大喇叭广播。信息在多人之间慢慢传开，计算量大幅降低。

### 面试怎么说

**Q：典型稀疏模式？**

A：滑动窗口（看附近 ±w）、步长（隔 d 个跳读）、全局哨兵（少数 token->全部）。组合使用。精度损失 1%~3%。

**Q：著名模型？**

A：Longformer、BigBird、Sparse Transformer。近年 FlashAttention 等全注意力优化更主流——与其少算不如更快算。

## 【46】RAG（检索增强生成）

### 用人话解释

模型的知识存在训练好的参数里，有两个致命问题：不更新（训练日之后的全不知道）、不精确（具体细节容易错乱）。RAG 的思路很简单：**回答前先翻书**。

用户提问题 → 先去知识库里搜相关内容 → 把搜到的文档块和问题一起喂给模型 → 模型基于检索到的资料来回答。这样既准确（基于真实文档）、又新鲜（知识库可以随时更新）、还能溯源（知道答案来自哪里）。

检索用嵌入模型（把文本编码成向量找相似的）+ BM25（关键词精确匹配），称为"混合检索"。目前 RAG 已是减少幻觉、做企业级问答的标准方案。

> 你可以这样记：RAG = 模型回答问题前先翻书查证，然后基于查到的资料来答。闭卷变开卷，答案可溯源，知识可更新。

### 面试怎么说

**Q：检索用什么模型？**

A：传统 BM25（关键词，不用训练），现代密集检索（嵌入模型，语义匹配）。最佳实践是混合检索——BM25 保精准 + 密集保语义。

**Q：怎么减少幻觉？**

A：闭卷变开卷。先查到权威资料再回答，不靠容易出错的内部记忆。检索靠谱可把幻觉率从 20~30% 降到个位数。

**Q：和长上下文窗口哪个好？**

A：互补。RAG 适合知识频繁更新、需明确出处。长窗口适合全面理解整个文档。可结合——RAG 查再用长窗口分析。
