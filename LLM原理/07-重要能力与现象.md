# 七、重要能力与现象

## 【33】上下文学习

**是什么**：不更新模型参数，仅通过 prompt 中给几个示例（few-shot），模型就能理解任务模式并对新输入做出正确响应。比如 prompt 里给两对"中文→英文"的翻译示例，再给一个新的中文句子，模型就能翻译出来。

**为什么这样设计**：上下文学习不是刻意设计的，而是"涌现"出来的——模型在预训练中读过大量包含"示例→应用"模式的文本（教程、FAQ、带例子的说明文），不知不觉学会了"看到几个例子 → 提炼规律 → 应用到新情况"。这个能力的深层魅力在于：它证明了 LLM 不是在做简单的模式匹配，而是在做某种程度的"元学习"（学习如何学习）。不同微调，微调把知识写入参数（永久改变），上下文学习是临时的（对话结束就忘）。

**和周边的联系**：上下文学习是涌现能力的代表。思维链是上下文学习的一个特殊形式（给的示例包含推理步骤）。系统提示也可以理解为"零示例的上下文学习"。

### 面试问答

**Q：上下文学习和微调的区别？**

A：上下文学习不动参数，靠 prompt 里的示例临时掌握任务。微调更新参数，把知识永久写入模型。前者像借来的技能，后者像学会的技能。

**Q：为什么会有这种能力？**

A：涌现出来的。>10B 后出现。可能源于预训练数据中大量的"示例→应用"模式，模型被迫学会了元学习。

**Q：示例顺序影响效果吗？**

A：影响很大，10%~30% 差异。由易到难效果好，最后一个示例影响最大。

## 【34】思维链

**是什么**：要求模型在输出最终答案前先写出中间推理步骤。比如不说"答案是 6"，而是说"5-2=3，3+3=6，所以答案是 6"。

**为什么这样设计**：复杂推理问题是"一步到位很难，拆开一步一步来很容易"。思维链把大跳跃拆成小跳跃——每个小跳跃（单步加减法、一个简单逻辑判断）都在模型的能力范围内，串起来就能解决复杂推理。更关键的是，中间步骤被写进上下文后成了"草稿纸"——模型在生成后续步骤时可以参考之前算出的结果，避免了一口气猜答案的盲区。需要模型 >10B 才有效，小模型不仅不能推理，还会写错误推理然后基于错误得出更差的答案。

**和周边的联系**：思维链是上下文学习 + 涌现能力的交叉产物。在 GSM8K 上，加了思维链的 GPT-4 从约 60% 提升到 92%。

### 面试问答

**Q：为什么思维链能提升推理？**

A：把复杂问题拆成多个简单步骤，每步模型都擅长。中间步骤写入上下文变成"草稿纸"，后续推理可以参考之前的结果。

**Q：零样本和少样本思维链的区别？**

A：零样本只需加"让我们一步一步思考"。少样本给 2~3 个带完整推理的示例。少样本效果更好，零样本胜在简单。

**Q：小模型能用吗？**

A：基本不行。小模型不仅不能正确推理，还会写错误推理再基于错误得出更糟答案。需 >10B 才有效。

## 【35】涌现能力

**是什么**：模型参数规模超过某个阈值后（通常数 B 到数百 B），突然出现的小模型完全不具备的复杂能力。不是性能的平滑提升，而是从"不会"到"会"的质变。

**为什么这样设计**：涌现不是故意设计的，是大规模训练后的自然结果。最被接受的解释是：某些复杂任务需要模型同时掌握多种子能力（理解语言、识别模式、多步推理），当模型太小时，每种子能力都不够好，组合起来就更差——所以表现为"完全不会"。当模型足够大，所有子能力同时达标，能力突然显现。有学术争议认为部分涌现只是"度量指标的非线性假象"——换平滑指标后曲线就是连续的。共识是：度量确实夸大了一些涌现，但核心涌现（上下文学习、思维链）是真实存在的。

**和周边的联系**：涌现能力和缩放法则紧密相连——缩放法则预测性能会提升，涌现解释了提升的形式。上下文学习和思维链是涌现的两个标志性例证。

### 面试问答

**Q：多大参数会出现？**

A：算术推理 8~13B，思维链 ~10B，零样本翻译 ~15B，幽默 50B+。不是一刀切，10~100B 是涌现集中区间。

**Q：举例哪些是涌现的？**

A：上下文学习、思维链推理、零样本翻译、代码生成、反讽理解。小模型全随机，过阈值突然可用。

**Q：是真的还是度量问题？**

A：有争议。部分涌现可能是度量假象（换指标就平滑了），但核心能力（上下文学习、思维链）的涌现真实存在。

## 【36】幻觉

**是什么**：模型生成的内容与事实不符——可能是与外部世界矛盾（说巴黎是德国首都），也可能与用户给的上下文矛盾（给的文章说 A 和 B 是朋友，模型总结成仇人）。

**为什么这样设计**：幻觉不是被设计的，是大语言模型的固有缺陷。根因在于：LLM 本质是"概率性的下一个词预测器"——它学会的是"在类似上下文中，人们通常会接什么词"，而不是"什么是真实的"。当模型内部知识不确定时，它会倾向于编造一个"听起来合理"的答案，因为训练数据中很少有"我不知道"的示例。温度越高、采样越随机，幻觉越严重。RAG（检索增强生成）是目前最有效的缓解手段——把生成从"闭卷考试"变"开卷考试"。

**和周边的联系**：幻觉是 LLM 需要对齐的核心理由之一。RAG 和 Function Calling 是两大抗幻觉技术。温度、Top-p 等解码策略只能轻微缓解，不是根本解。

### 面试问答

**Q：幻觉有哪些类型？**

A：事实性幻觉——跟真实世界不符；忠实性幻觉——跟用户给的上下文矛盾。后者在 RAG 场景尤其危险——给了资料还乱编。

**Q：怎么减轻？**

A：解码侧治标（降温度、调 Top-p）。根本方案是 RAG（先检索再基于资料答）、Function Calling（调工具获真相）、更好的对齐训练。

**Q：RAG 为什么能减幻觉？**

A：闭卷变开卷。没有 RAG 靠内部记忆（可能记错），有 RAG 先查权威资料再答。检索靠谱的话幻觉率可从 20~30% 降到个位数。

## 【37】对齐

**是什么**：让模型的行为符合人类价值观和期望——通常聚焦三个维度：有用（Helpful）、诚实（Honest）、无害（Harmless），简称 HHH。

**为什么这样设计**：预训练模型学会了"语言"和"知识"，但它不知道什么是"对的"。问"怎么做炸弹"，它能从预训练数据里找到真实的制作方法并回答——对"下一个词预测"这个目标来说它做得很好，但对人类来说这是灾难。对齐就是给模型加上道德和行为边界。RLHF 和 DPO 是目前最主流的技术——它们用人类的偏好数据告诉模型"这种回答更好，那种更差"。HHH 三维度有时冲突——"有用"和"无害"常常需要权衡，对齐就是在这些张力中找到正确的平衡。

**和周边的联系**：对齐是训练流程的最后一步（在预训练和 SFT 之后），用 RLHF 或 DPO 实现。幻觉问题、安全问题都通过对齐来解决。

### 面试问答

**Q：HHH 是什么？**

A：Helpful（有用）、Honest（诚实）、Harmless（无害）。三维度有时冲突，对齐就是在张力中找到正确平衡。

**Q：对齐和微调有什么不同？**

A：微调关注能力（任务表现），对齐关注行为边界（什么该说什么不该说）。SFT 提供能力，RLHF/DPO 提供边界。

**Q：主流对齐技术？**

A：RLHF（奖励模型+PPO）、DPO（直接用偏好数据）、Constitutional AI（预定义规则+自我审查）、红队测试。趋势是组合使用。

## 【38】缩放法则

**是什么**：模型性能（测试损失）与参数量 N、训练数据量 D、计算量 C 之间呈幂律关系：Loss ∝ N^(-α) + D^(-β) + 常数。意味着把模型和数据等比放大，性能会可预测地提升。

**为什么这样设计**：缩放法则不是设计，而是通过大量实验总结出的经验规律。它回答了一个重要的战略问题——"值得砸更多钱做大模型吗？"。答案是：值得，但要 N、D、C 三者等比增长，光堆参数不给够数据是浪费（Chinchilla 法则：参数和训练 token 约 1:20 最优）。近年来边际效益递减的问题让共识从"盲目做大"转向"聪明地做大"——更好数据、更优架构（MoE）、更高效训练策略同样能榨出性能。

**和周边的联系**：缩放法则解释了为什么要做大规模训练，涌现能力解释了规模带来的质变。MoE 是一种"偏离缩放法则"的办法——计算量不大但参数量巨大。

### 面试问答

**Q：核心结论？**

A：Loss ∝ 1/N^α + 1/D^β。模型和数据等比放大 → 性能可预测提升。但边际效益递减。

**Q：参数、数据、计算三者怎么配？**

A：Chinchilla 法则——参数和 token 约 1:20 最优。之前模型（GPT-3）参数太多数据太少。实际看优化目标——推理成本敏感就用更多数据训更小模型。

**Q：越大一定越好吗？**

A：边际效益递减+成本指数增长。现在共识："聪明地做大"——更优数据、更高效架构（MoE）、更好策略。
