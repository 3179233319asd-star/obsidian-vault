# 七、重要能力与现象

## 【33】上下文学习

### 用人话解释

上下文学习就是"给几个例子，模型就学会了新任务，不需要改参数"。比如你在 prompt 里给两组"中文→英文"的翻译示例，再给一个新的中文句子，模型就直接输出翻译——它从没见过"翻译任务"的专门训练数据，但仅凭两个示例就明白了要做什么。

这能力不是被设计出来的，是"涌现"的——模型小于 10B 参数时基本不会，大到一定程度突然就开窍了。最合理的解释是：预训练数据里有大量"给例子→引出一个规律→应用到新情境"的模式（教材、FAQ、带例子的说明文），模型在大量阅读中被迫学会了"看例子提炼规律"这个元技能。

> 你可以这样记：上下文学习 = 现场教学。给模型几个范例它就能照做，不用改任何参数。这是大模型"聪明"的核心体现，小模型做不到。

### 面试怎么说

**Q：和微调的本质区别？**

A：上下文学习不动参数，靠 prompt 里给的示例临时学会，对话结束就忘。微调把知识写入参数，永久改变。前者像借来的技能，后者像学会的技能。

**Q：为什么有这个能力？**

A：涌现出来的。>10B 才明显。可能源于预训练数据中大量"示例→规律→应用"的模式，模型被迫学会了元学习。

**Q：示例顺序影响效果吗？**

A：影响很大（10%~30% 差异）。由易到难好，最后一个示例影响最大。

## 【34】思维链

### 用人话解释

如果直接让模型回答一个数学题"小明有 5 个苹果给了 2 个又买了 3 个，现在有几个？"，它可能直接猜一个数。但如果你让它"先一步一步写推理过程，再给答案"，准确率就从 60% 飙升到 90% 以上。

原理：复杂推理一步到位很难，但拆成"5-2=3，3+3=6，所以答案是 6"这种小步骤，每步都很简单。更关键的是——中间步骤写出来之后变成了"草稿纸"，模型在生成后续步骤时可以回看前面算出的结果。

这也是涌现能力，小模型（<10B）不仅不能推理，还会写出错误推理然后基于错误得出更离谱的答案。

> 你可以这样记：思维链 = 让模型在草稿纸上列竖式。别一口气猜答案，分步写出来，每一步都简单，串起来能解难题。大模型才有效。

### 面试怎么说

**Q：为什么能提升推理？**

A：复杂问题拆成多个简单小步，每步模型都擅长。中间步骤写进上下文变成草稿纸，后续可参考前面结果。

**Q：零样本和少样本？**

A：零样本只加一句"让我们一步一步思考"。少样本给 2~3 个带完整推理的范例。少样本效果更好，零样本胜在简单。

**Q：小模型能用吗？**

A：不行。小模型会写错误推理再基于错误得离谱答案。需 >10B。

## 【35】涌现能力

### 用人话解释

小模型（1B 参数）做算术推理 ≈ 随机猜测，10B 模型突然就能算了。不是"越来越好"的平滑渐变，而是从"完全不会"到"会用"的跳变。这就是涌现能力——模型大到一定程度后突然出现的能力。

最被接受的解释是：复杂任务需要多种子能力同时达标。当模型太小时每种子能力都半桶水，组合在一起还是半桶水。当模型大到所有子能力同时过关后，能力才突然显现。目前有学术争议认为部分涌现可能是"度量指标的假象"（换成平滑指标就连续了），但共识是核心涌现（上下文学习、思维链、零样本翻译）真实存在。

> 你可以这样记：涌现 = 量变引起质变。模型从小变大，能力不是慢慢涨分的，而是突然"开窍"的。就像足够多的神经元连在一起突然产生了意识。

### 面试怎么说

**Q：多大参数会出现？**

A：不同任务门槛不同。算术推理约 8~13B，思维链约 10B，零样本翻译约 15B。10~100B 是集中涌现区间。

**Q：哪些是涌现的？**

A：上下文学习、思维链推理、零样本翻译、代码生成、反讽理解。小模型全随机，过阈值突然可用。

**Q：真的存在还是度量假象？**

A：有争议。部分涌现确实是度量造成的假象，但核心能力（上下文学习、思维链）的涌现真实存在。

## 【36】幻觉

### 用人话解释

模型有时候会信心满满地说出完全错误的信息——"巴黎是德国的首都""'1+1'的泰勒展开结果是 42"。这就是幻觉。

根因在于：LLM 本质是"概率性的下一个词预测器"。它学的是"在类似语境下人们通常会接什么词"，而不是"什么信息是真的事实"。当模型对一个问题不确定时，它不会说"我不知道"（因为训练数据中很少有这种示范），而是倾向于编造一个"听起来合理"但实际不对的答案。

RAG（先查资料再答题）和 Function Calling（打电话问专业工具）是目前最有效的抗幻觉手段。把"闭卷考试"变成"开卷考试"。

> 你可以这样记：幻觉 = 模型爱"不懂装懂"。因为它的训练目标是"说话像人"，不是"说真话"。对付方法就是让它考试时能翻书（RAG）或能打电话问（Function Calling）。

### 面试怎么说

**Q：有哪些类型？**

A：事实性幻觉——跟真实世界矛盾。忠实性幻觉——跟用户给的上下文矛盾。后者在 RAG 场景尤其危险：给了参考材料还乱编。

**Q：怎么减轻？**

A：解码侧治标（降温度调 Top-p）。根本方案：RAG（先查再答）、Function Calling（调工具获取真相）、更好的对齐训练。

**Q：RAG 为什么有效？**

A：闭卷变开卷。模型不靠不可靠的内部记忆，先查权威资料再答。检索靠谱的话幻觉率可从 20~30% 降到个位数。

## 【37】对齐

### 用人话解释

预训练模型会续写文本，但它没有"什么该说什么不该说"的概念。你问"怎么做炸弹"，它能从网络数据里学到真实的制作流程并回答——对"预测下一个词"这个任务来说它做得很好，但对人类来说这是灾难。

对齐就是给模型加上道德和行为边界。技术上用 RLHF 或 DPO 实现：让人类标注"哪种回答更好"，再用这些偏好数据微调模型。三个核心目标简称 HHH——有用（Helpful，解决用户问题）、诚实（Honest，不确定就说不知道）、无害（Harmless，不做危险事）。这三个目标有时彼此冲突，对齐就是在张力的夹缝中找到正确平衡。

> 你可以这样记：对齐 = 给通才上一堂"怎么做个好人"的课。预训练让模型会说话，对齐让它说"对的话"。HHH 是用、诚、无害三者兼顾。

### 面试怎么说

**Q：HHH 是什么？**

A：Helpful（有用）、Honest（诚实）、Harmless（无害）。三维度有时冲突——对齐就是在冲突中找到正确权衡。

**Q：和微调有什么不同？**

A：微调关注"能力"（在任务上表现好），对齐关注"行为边界"（该说什么不该说什么）。SFT 给能力，RLHF/DPO 给边界。

**Q：主流技术？**

A：RLHF、DPO、Constitutional AI、红队测试。趋势是组合使用多种方法。

## 【38】缩放法则

### 用人话解释

缩放法则回答一个功利的问题：多花钱把模型做大，效果能提升多少？答案是可预测的——模型性能与参数量 N、数据量 D、计算量 C 呈幂律关系。简单说：模型翻倍 → 错误率约降为原来的 0.9 倍。

但有前提：三者要等比增长。光堆参数数据不够是浪费。Chinchilla 论文的经典结论：参数数和训练 token 数约 1:20 最优。最近边际效益递减的趋势让共识从"盲目做大"转向"聪明地做大"——更好的数据质量、更高效的架构（MoE）、更优的训练策略。

> 你可以这样记：缩放法则 = 大模型投资指南。告诉你要投入多少参数、多少数据、多少算力，才能换来预期效果。不是越大越好，要按比例配。

### 面试怎么说

**Q：核心结论？**

A：Loss ∝ 1/N^α + 1/D^β。等比例同时放大参数和数据 → 性能可预测提升。但边际效益递减。

**Q：三者怎么配？**

A：Chinchilla 法则——参数和 token 约 1:20 最优。实际情况看优化目标——推理成本敏感就用更多数据训更小模型。

**Q：越大一定越好吗？**

A：边际效益递减 + 成本指数增长。现在共识："聪明地做大"——更优数据、更高效架构、更优策略。
