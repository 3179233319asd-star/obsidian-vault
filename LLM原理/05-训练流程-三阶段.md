# 五、训练流程（三个阶段）

## 【21】预训练

### 用人话解释

预训练就是把一个"什么都不懂"的模型，用海量文本（互联网网页、书籍、代码等）训练成一个"知识渊博的通才"。任务就是反复做"预测下一个词"（CLM），数据量以万亿 token 计，需要几千张 GPU 跑几个月，占整个训练成本的 99% 以上。

预训练之后，模型已经学会了语法、语义、常识推理、多种语言……但它不会聊天。你问它"今天天气怎么样"，它可能给你续写一段互联网风格的文本，而不是像个助手一样回答。这就是为什么还需要后续的 SFT 和 RLHF——预训练给你一个"聪明的脑子"，后续微调教它"怎么跟人交流"。

> 你可以这样记：预训练 = 通识教育，花钱最多但一次投入永久受益。出来的基座模型什么都懂一点，但还没有被教过"该怎么说话"。

### 面试怎么说

**Q：数据量和来源？**

A：万亿 token 级。网页（Common Crawl）是大头，加上书籍、论文、代码、维基百科。质量比数量重要。

**Q：学到的是死记硬背吗？**

A：不是。学的是统计规律——语法、语义、推理模式。能在新句子上泛化。但确实对高频内容有记忆现象。

**Q：为什么非要花巨资做预训练？**

A：预训练培养一个通才（一次投入），之后通过微调快速适配几百种任务（便宜可重复）。每个任务都从零训练，成本没法承受。

## 【22】数据标注

### 用人话解释

预训练让模型学会了"语言"，但它不知道"什么样的回答方式人类更喜欢"。数据标注就是人写参考答案给模型看——你写几千条"用户这样问，你应该这样答"的例子，模型就能举一反三学会对话风格。

标注数据虽然量不大（通常几万条），但对模型行为影响巨大。因为微调时学习率很小，数据量也不大，每一条的权重都很高。几千条高质量标注 > 十万条粗糙标注。

> 你可以这样记：数据标注 = 给模型看"标准答案长什么样"。量不大但每一条都很关键——标注质量直接决定微调后模型的行为。

### 面试怎么说

**Q：哪些阶段需要标注？**

A：SFT 需要指令-回答对（最贵，需创造性写作）。RLHF 需要偏好排序（相对便宜，选择题）。评估也需要人工验证。

**Q：标注质量怎么影响模型？**

A：质量 = 天花板。预训练教"能说话"，标注教"该说什么"。几千条高质量 > 十万条水货。

**Q：能减少标注量吗？**

A：用强模型生成候选答案人工修改、先大量低质再少量高质对齐。但 1~5 万条高质量标注仍是 SFT 标配。

## 【23】有监督微调（SFT）

### 用人话解释

SFT 就是拿着人工标注的"指令-回答"对，继续训练预训练模型。技术上还是做"预测下一个词"，但有几处关键差异：只计算回答部分的损失（指令部分不算），学习率比预训练小 10~100 倍。

为什么学习率要这么小？因为预训练积累的通用知识太宝贵了，微调只是"调行为风格"，不是重新学习知识。学习率太大 = 把底子洗掉 = 灾难性遗忘。

SFT 是让基座模型变成"会聊天的助手"的关键一步。经过 SFT 的模型已经能很好遵循指令了，但回答可能还不安全或不完全符合人类价值观——这就留给下一步 RLHF/DPO 解决。

> 你可以这样记：SFT = 岗位入职培训。基座模型是博学但不会聊天的通才，SFT 用几千条范例教它"当用户这样问时你必须这样回应"。学习率很小，因为不能让培训内容覆盖通识教育。

### 面试怎么说

**Q：SFT 和预训练的核心区别？**

A：数据上 SFT 用人工写的指令-回答对，预训练用无标注文本。目标上 SFT 学对话格式，预训练学通用能力。SFT 学习率比预训练小 10~100 倍。

**Q：微调学习率为什么这么小？**

A：预训练积累了大量知识，微调只是调整行为风格。学习率太大会把底子洗掉。

**Q：需要多少数据？**

A：几千到几万条就能会聊天。质量 > 数量，多样性 > 单任务规模。Alpaca 用了 5.2 万条，Llama 2-Chat 用了 2.7 万条。

## 【24】RLHF（人类反馈强化学习）

### 用人话解释

SFT 之后的模型会聊天了，但可能会说出不安全或不恰当的内容。RLHF 是让模型的回答更符合人类偏好的方法，分三步：

1. **收集偏好数据**：让模型对同一问题生成多个回答，人工标注哪个更好（A > B）
2. **训练奖励模型**：用这些偏好数据训练一个"打分器"，学会像人类一样评判回答好坏
3. **PPO 强化学习**：用奖励模型的打分作为信号，微调语言模型去生成高分回答

为什么需要中间的奖励模型？因为强化学习需要海量的打分（每次生成都要扣分），不可能每次都请真人打分。奖励模型就是人类评判标准的"自动版"。PPO 负责控制更新幅度——防止模型为了刷分而输出一堆不像人话的高分回答。

> 你可以这样记：RLHF = 人工选美 → 训练自动评委 → 让模型对着评委练到评委满意。PPO 是保险绳，防止模型为了讨好评委而练歪了。

### 面试怎么说

**Q：三步是什么？**

A：1) 收集人类偏好排序；2) 训练奖励模型模仿人类打分；3) 用 PPO 强化学习优化语言模型去拿高分。奖励模型是人类的"可微分替代品"。

**Q：PPO 起什么作用？**

A：限制每次更新幅度，防止模型"刷分"过头变得离谱。在"变好"和"不乱变"之间找平衡。

**Q：RLHF 缺点？**

A：需同时维护三个模型（工程复杂）、奖励标注主观（不同人偏好不同）、奖励黑客（模型钻奖励模型空子）。DPO 为解决这些而提出。

## 【25】DPO（直接偏好优化）

### 用人话解释

RLHF 太复杂了——要训练奖励模型、跑强化学习、调一堆超参，而且训练不稳定。DPO 的思路是"既然数学上能证明 RLHF 目标有最优解公式，那直接代入就行了"。

具体做法：直接从偏好数据（chosen vs rejected）出发构造损失函数——提高 chosen 回答的概率，同时降低 rejected 回答的概率，再保持离参考模型不太远。整个过程不需要奖励模型、不需要强化学习，只需几行代码改动损失函数。训练稳定性和收敛速度远超 RLHF。

> 你可以这样记：DPO = RLHF 的极简版。把"人工偏好 → 奖励模型 → PPO"这三步压缩成一步——直接用偏好数据优化语言模型。简单、稳定、效果好。

### 面试怎么说

**Q：怎么省掉奖励模型的？**

A：RLHF 优化目标存在数学闭式解。DPO 直接代入，得到只依赖语言模型本身的损失函数。不需要奖励模型和 PPO。

**Q：DPO 损失的直觉？**

A：对每组偏好对——提高 chosen 概率，降低 rejected 概率，同时别离参考模型太远。一句话：朝标注方向走，别走太远。

**Q：DPO 优劣？**

A：优势：实现简单（几行代码）、训练稳定、超参少。劣势：假设偏好为 Bradley-Terry 模型，复杂偏好结构下 PPO 可能更灵活。
