# 五、训练流程（三个阶段）

## 【21】预训练

**是什么**：在海量无标注文本上做 CLM（预测下一个词）训练，让模型学会语言规律、世界知识和推理能力。这是 LLM 训练的基石，占整个训练算力和成本的 99%+。

**为什么这样设计**：预训练就像通识教育——你不需要一个专门的"数学教材"，只需要让模型读足够多的文本，它自己就能从数据中归纳出算术运算的规律。预训练后的基座模型已经是一个"知识渊博但不太会聊天的通才"，后续微调只需教它对话格式和人类偏好。这个分工的经济意义巨大：预训练贵但一次投入、永久受益；微调便宜、可重复做、适配不同场景。

**和周边的联系**：预训练用的是 CLM 任务和自监督学习范式。预训练的数据量和质量直接决定模型的能力天花板。缩放法则指导预训练的资源分配。

### 面试问答

**Q：预训练的数据量和来源？**

A：万亿 token 级（LLaMA 2 约 2 万亿）。来源：网页（Common Crawl，占大头）、书籍、论文、代码、维基百科。质量比数量重要。

**Q：模型学到的是什么？死记硬背吗？**

A：学的是统计规律——语法、语义、推理模式，不是背诵。能在从未见过的句子上泛化。但高频内容确实有记忆现象。

**Q：为什么不直接微调，要做昂贵的预训练？**

A：预训练是培养通才（一次投入），微调是岗位培训（便宜可重复）。每个任务都从零训练成本无法承受。

## 【22】数据标注

**是什么**：人工为训练数据写参考答案或做偏好判断。SFT 需要人写"理想回答"，RLHF 需要人评判"A 和 B 哪个更好"，评估需要人做质量把关。

**为什么这样设计**：模型从预训练中学到了"语言是什么样"和"世界上有哪些知识"，但它不知道"什么样的回答方式是人类喜欢的"。数据标注就是告诉模型这个信息——你给模型看几千个"用户这样问，你应该这样答"的例子，它就能举一反三。标注质量直接影响模型行为的走向，因为微调阶段学习率很小、数据量也不大，每一条标注的权重都很大。

**和周边的联系**：标注数据直接用于 SFT 和 RLHF。标注质量 = 微调效果的天花板。业界趋势是用强模型（GPT-4）生成初稿人工修改，降本增效。

### 面试问答

**Q：哪些阶段需要标注？**

A：SFT 需要指令-回答对（最贵，需创造性写作）。RLHF 需要偏好排序（相对便宜，选择题）。评估也需要人工验证。

**Q：标注质量怎么影响模型？**

A：质量 = 天花板。预训练教"能说话"，标注教"该说什么"。标注错误 → 模型学到错误模式。几千条高质量好于十万条水货。

**Q：能减少标注量吗？**

A：用强模型生成候选答案人工挑选修改；先大量低质再少量高质对齐。但 1~5 万条高质量人工标注仍是 SFT 标配。

## 【23】有监督微调（SFT）

**是什么**：用人工标注的（指令，理想回答）对，继续训练预训练好的模型。技术上和预训练一样做"预测下一个词"，但只计算回答部分的损失，且学习率小很多。

**为什么这样设计**：预训练模型虽然会预测下一个词，但它不知道该预测什么样的词——是继续互联网风格的文本，还是像助手一样回答。SFT 通过人工范例告诉模型"当用户这样说时，你应该这样回应"。学习率设得很小（比预训练小 10~100 倍），因为预训练积累的通用知识已经非常宝贵，微调只是"调风格"，不能把底子洗掉。

**和周边的联系**：SFT 是连接"基座模型"和"可用的对话助手"的关键桥梁。SFT 是必要但不充分——后续还需要 RLHF/DPO 解决安全对齐问题。

### 面试问答

**Q：SFT 和预训练的核心区别？**

A：预训练数据 = 无标注文本，SFT 数据 = 人工写的指令-回答对。预训练学通用能力，SFT 学对话格式。技术上都做下一词预测，但 SFT 学习率小 10~100 倍。

**Q：微调学习率为什么比预训练小？**

A：预训练已经积累了大量知识，微调只是调行为风格。学习率太大会把预训练学到的知识洗掉。

**Q：需要多少数据？**

A：几千到几万条就能让模型会聊天。Alpaca 5.2 万条，Llama 2-Chat 2.7 万条。质量 >> 数量，多样性 >> 单任务规模。

## 【24】RLHF（人类反馈强化学习）

**是什么**：让模型输出更符合人类偏好的三阶段方法：1) 收集人类对模型不同回答的偏好排序；2) 用这些数据训练奖励模型（学会模仿人类打分）；3) 用 PPO 强化学习微调语言模型以最大化奖励分数。

**为什么这样设计**：SFT 之后的模型会对话了，但回答可能不安全、不诚实、不符合人类价值观。RLHF 提供了一个精细的优化信号——"你这个回答好，那个回答不好"。之所以需要奖励模型这个中间人，是因为强化学习需要海量反馈（每步生成都要打分），不可能每次都请人类来打。奖励模型就是人类偏好的可微分替代品。PPO 算法在此处增加了约束：你不能为了拿高分就变得跟原始模型差别太大（否则可能乱说），每次更新幅度要有限制。

**和周边的联系**：RLHF 是 ChatGPT 成功的关键创新。DPO 是 RLHF 的简化替代。RLHF 成功的前提是 SFT 已经做好了基本的指令遵循能力。

### 面试问答

**Q：RLHF 的三步是什么？**

A：1) 收集人类偏好排序；2) 训练奖励模型学会模仿人类打分；3) PPO 强化学习让语言模型最大化奖励。奖励模型是人类的"可微分替代品"。

**Q：PPO 起什么作用？**

A：约束更新幅度——防止模型为了拿高分变得太离谱（奖励过拟合），在"变好"和"不乱变"之间找平衡。

**Q：RLHF 有什么缺点？**

A：需同时维护三个模型，工程复杂；奖励标注主观；奖励黑客（模型钻奖励模型空子）。DPO 为解决这些痛点而生。

## 【25】DPO（直接偏好优化）

**是什么**：跳过 RLHF 的奖励模型和 PPO 环节，直接用偏好数据（chosen vs rejected）构造损失函数来优化语言模型。数学上，DPO 推导了 RLHF 目标的最优解闭式形式，将问题转化为标准的有监督训练。

**为什么这样设计**：RLHF 太复杂了——要训练奖励模型、跑 PPO 强化学习、调一堆超参数，而且训练很不稳定。DPO 的核心洞察是：RLHF 中"最大化奖励同时不偏离太远"这个问题，在理论上可以直接写出最优解的公式。把它代回去，就得到一个简洁的损失函数——对每组偏好对，提高 chosen 回答的生成概率，降低 rejected 回答的生成概率，同时保持跟参考模型的距离不要太远。实现只需改损失函数（几行代码），训练稳定性和收敛速度远超 RLHF。

**和周边的联系**：DPO 直接替代 RLHF，数据也是偏好对（chosen/rejected）。目前两者并存——DPO 简单稳定但理论假设强，RLHF 灵活但复杂。二者都需要先完成 SFT。

### 面试问答

**Q：DPO 怎么省掉奖励模型的？**

A：RLHF 优化目标存在闭式解。DPO 直接代进去，得到一个只依赖语言模型本身的损失函数。不需要训练奖励模型，不需要 PPO。

**Q：DPO 损失的直觉？**

A：对每组偏好对——提高 chosen 概率，降低 rejected 概率，同时保持离参考模型不太远。一句话：朝标注方向走，但别走太远。

**Q：DPO 的优劣？**

A：优势：实现简单，训练稳定，超参数少。劣势：假设偏好为 Bradley-Terry 模型，复杂偏好结构下 PPO 更灵活。
