# 十、评估方法

## 【47】困惑度

**是什么**：交叉熵损失的指数形式：PPL = exp(Loss)。可以理解为"模型在每个位置上平均要从多少个候选词中做选择"——PPL=50 就像在面对一个 50 词的虚拟词表做选择题。

**为什么这样设计**：交叉熵损失本身的值比较抽象（0.05 跟 0.1 差距多大？），不适合人类直观感受。困惑度做指数变换后，单位变得可解释——"等效词表大小"。但困惑度只测量"预测准确性"，不衡量生成质量的其他维度（多样性、事实正确性、逻辑连贯性）。一个模型可以困惑度很低（非常自信地重复常见词），但生成内容全是废话。所以困惑度适合训练监控和同架构对比，不适合做生成质量的最终评判。

**和周边的联系**：困惑度直接来源于交叉熵损失函数。是预训练阶段最主要（有时唯一）的监控指标。不同词表的模型困惑度不可直接比较。

### 面试问答

**Q：困惑度和交叉熵的关系？**

A：困惑度 = exp(交叉熵)。交叉熵是每个 token 的不确定度，困惑度转换成"等效词表大小"——PPL=50 意味着每步平均在 50 个候选词中选。

**Q：困惑度低就是好模型吗？**

A：不一定。只测预测准确，不测多样、连贯、事实正确。模型可以 PPL 很低但全是废话。适合训练监控，不适合最终质量评判。

**Q：多少算好？**

A：看词表和数据领域，没绝对值。GPT-2 在 WikiText 约 30~35，GPT-3 约 20。数值差距不大对应的实际能力差距往往很大。

## 【48】常用基准（MMLU / GSM8K / HumanEval）

**是什么**：三套标准化的模型能力测试——MMLU（57 学科多选题，测知识广度）、GSM8K（小学数学应用题，测推理深度）、HumanEval（编程题，测代码生成能力）。

**为什么这样设计**：不同模型放在同一个"考场"里公平比较。MMLU 覆盖人文社科理工医法，5-shot 评估，GPT-4 约 86%。GSM8K 每道题需 2~8 步推理，答案是纯数字评判客观，GPT-4 无思维链约 60%、用思维链 92%——完美量化了推理链的价值。HumanEval 用 pass@k 指标——pass@1 = 首次生成就对的概率，pass@100 = 生成 100 次至少对一次。业界最看重 pass@1，因为实际使用不可能每题生成 100 次。

**和周边的联系**：这些基准直接驱动了思维链、推理优化等技术的发展。新模型发布必报 MMLU 和 HumanEval 分数。但过度优化 benchmark 也可能导致"刷榜"现象。

### 面试问答

**Q：MMLU 怎么评？**

A：57 学科，四选一，5-shot。GPT-4 约 86%。目前最重要的通用知识考试，新模型发布必报。

**Q：GSM8K 为什么能测推理？**

A：小学数学应用题，需 2~8 步推理。数字换了得重算，没法靠记忆。答案纯数字，客观评判。

**Q：HumanEval 的 pass@k 什么意思？**

A：164 道编程题，生成 k 次至少对一个。pass@1 = 一次就对的概率（可靠性），业界最看重。

## 【49】人工评估

**是什么**：由人类对模型输出进行质量打分或偏好选择，通常从有用性、诚实性、无害性、连贯性等多维度评判。是评估的黄金标准，因为自动指标无法捕捉回答的"好不好"。

**为什么这样设计**：自动指标只能比"像不像"（字面匹配度），但人类关注的是"对没对、好不好"。同样的意思，不同说法在自动指标里得分可能完全不同，但对人类来说等价。Arena-style 盲评是目前最被认可的方式——把两个模型的回答并列（不标哪个是哪个），让人类投票选更好的。为了保证可靠性，需要双盲实验、标准化评分手册（每分数有锚定示例）、多人独立打分取平均（至少 3 人）、一致性检验——每个环节都会增加成本，但结果的可靠性全靠这些。

**和周边的联系**：人工评估的结果直接反馈给对齐策略——发现模型在哪方面不符合人类期望，就针对性收集偏好数据做 RLHF/DPO。和自动基准互为补充。

### 面试问答

**Q：为什么自动指标不够？**

A：自动指标只比字面匹配度。同样意思不同表达得分完全不同，风格、幽默、同理心完全无法自动量化。人类能感知细微差别。

**Q：评估哪些维度？**

A：准确性、有用性、诚实性、无害性、信息密度、指令遵循度。维度看场景——客服看解决率，创意看趣味性，代码看通过率。

**Q：怎么减少偏差？**

A：双盲、标准化评分手册（锚定示例）、多人独立打分（至少 3 人）、统计检验、随机打乱顺序防位置效应。成本高但可靠性全靠这些细节。
